# Feat 文档写作标准 - 参考手册

本文档为 SKILL.md 的补充，提供详细的结构模板、检查清单与示例库引用。

## 四大文档类型详细模板

### 教程 (Tutorials)

**目标**：让读者从零开始完成一个完整的学习目标，建立信心。

**黄金法则**：
- 前 50% 内容内必须有可运行的代码示例
- 每完成一步都有明确的验证方式（能看到效果）
- 避免解释原理，专注「怎么做」

**推荐结构**：
```markdown
## [动词+目标，如：构建你的第一个 HTTP 服务器]

### 你将学到
- 学会 xxx
- 掌握 yyy
- 实现 zzz

### 前置要求
- JDK 8+
- Maven 3.6+
- （其他依赖）

### 步骤 1：xxxx
[简要说明这一步做什么]

```java title="文件名.java"
// 完整可运行代码
```

[验证方式：如何确认这一步成功]

### 步骤 2：xxxx
...

### 步骤 N：验证结果
[完整的测试方法，确保读者能确认成功]

### 总结回顾
- 我们完成了什么
- 关键概念回顾
- 下一步学习建议
```

**人称**：多用「你」指导，适当「我们」共学。亲切友好，像导师带学生。

---

### 操作指南 (How-to Guides)

**目标**：解决一个具体问题，提供多种方案供选择。

**黄金法则**：
- 问题描述 → 推荐方案 → 替代方案 → 注意事项
- 至少提供一种完整可运行的解决方案
- 包含常见错误和排错方法

**推荐结构**：
```markdown
## 如何 [具体任务]

### 问题描述
[什么场景下会遇到这个问题]

### 推荐方案
[最常用/最佳的解决方案]

```java
// 完整示例代码
```

**配置说明**：
| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| xxx | 是 | - | xxx |

### 替代方案
#### 方案 A：[名称]
[适用场景]：xxx
[优点]：xxx
[缺点]：xxx

#### 方案 B：[名称]
...

### 注意事项
- ⚠️ 常见问题 1
- ⚠️ 常见问题 2

### 相关链接
- [链接到相关教程]
- [链接到 API 参考]
```

**人称**：主要「你」，少用「我们」。专业直接，像技术支持。

---

### 概念解释 (Explanations)

**目标**：帮助读者理解「为什么这样设计」「底层原理是什么」。

**黄金法则**：
- 用类比解释抽象概念（如 NIO 类比餐厅服务）
- 结合架构图或流程图说明
- 分析设计权衡（为什么选 A 而不是 B）

**推荐结构**：
```markdown
## [主题] 解析 / 机制 / 原理

### 背景和动机
[为什么会出现这个概念/技术]
[解决了什么问题]

### 核心设计思想
[用最通俗的语言解释设计理念]
[适当的类比帮助理解]

### 实现原理
```mermaid
[架构图/流程图/时序图]
```

[关键代码片段说明原理，非完整示例]

### 性能优势与权衡
| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| A | ... | ... | ... |
| B | ... | ... | ... |

### 扩展思考
- 与其他技术的对比
- 演进历史
- 最佳实践建议
```

**人称**：多用「我们」引导思考，少用「你」。客观理性，像技术分享。

### 参考文档 (References)

**目标**：快速查阅 API、参数、返回值等精确信息。

**黄金法则**：
- 信息密度高，便于快速查找
- 每个参数都有类型、默认值、说明
- 简短准确的示例（不需要完整 main）

**推荐结构**：
```markdown
## [组件/API] 参考

### API 概览
[一句话描述这个组件的作用]

```java
// 典型的使用模式
Component c = new Component();
c.method();
```

### 构造方法

#### `ComponentName(Type param)`
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| param | Type | 是 | - | 参数说明 |

### 配置方法

#### `methodName(Type param)`
[方法作用简述]

**参数**：
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| param | Type | 否 | null | 参数说明 |

**返回值**：ReturnType - 返回值说明

**示例**：
```java
// 简短示例，展示用法即可
component.method(value);
```

### 异常处理
| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| XxxException | 当...时抛出 | 应该... |

### 版本差异
| 版本 | 变更 |
|------|------|
| v1.4.0 | 新增 xxx 方法 |
| v1.3.0 | 废弃 yyy 方法 |
```

**人称**：第三人称，客观中立。简洁精确，像官方手册。

## 代码示例规范

### 示例来源优先级

1. **首选**：`feat-test` 模块中的真实可运行代码
2. **次选**：`feat-ai`、`demo/` 目录中的项目内示例
3. **补充**：为文档专门编写的精简示例（但必须可运行）

### 代码注释模板

根据文档类型使用不同的注释头：

**教程**：
```java
/**
 * 教程：构建 HTTP 服务器
 * 学习目标：理解 Feat 基本用法
 * 预期效果：访问 localhost:8080 显示 "Hello World"
 * 
 * @see feat-test/src/main/java/tech/smartboot/feat/demo/HelloWorld.java
 */
```

**操作指南**：
```java
/**
 * 操作指南：配置 HTTPS 支持
 * 适用场景：生产环境需要 SSL 加密
 * 注意事项：需要有效的证书文件
 * 
 * @see feat-test/src/main/java/tech/smartboot/feat/demo/HttpsPemDemo.java
 */
```

**概念解释**：
```java
/**
 * 概念解释：异步处理机制
 * 目的：演示 CompletableFuture 的使用方式
 * 
 * @see feat-test/src/main/java/tech/smartboot/feat/demo/AsyncHttpDemo.java
 */
```

**参考文档**：
```java
/**
 * HttpServer.httpHandler(handler) 方法示例
 * @param handler 请求处理器
 * @return HttpServer 实例（链式调用）
 * @throws IllegalStateException 如果服务器已启动
 */
```

### JDK 8 兼容性速查

✅ **允许使用**：
- Lambda 表达式：`() -> {}`
- Stream API：`list.stream().filter(...).collect(...)`
- 方法引用：`String::valueOf`
- 接口默认方法
- 传统字符串拼接 `"a" + b`

❌ **禁止使用**（JDK 9+ 特性）：
- 文本块 `"""..."""`
- `var` 关键字
- Record 类型
- 增强 switch
- `Optional.isEmpty()`
- 任何 JDK 9+ 特性

## 文档体系化写作指南

### 知识分层原则

为避免文档间内容重复、形成割裂，需遵循以下分层原则：

```
┌─────────────────────────────────────────────────────────────┐
│  第 1 层：入门引导（唯一）                                    │
│  • cloud/getstart.mdx - Feat Cloud 快速入门                  │
│  • server/getstart.mdx - Feat Server 快速入门                │
│  职责：首次接触，建立整体认知，不涉及细节                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  第 2 层：功能教程（按功能拆分）                               │
│  • cloud/db.mdx - MyBatis 集成                               │
│  • cloud/controller.mdx - Controller 开发                    │
│  职责：带读者完成一个完整功能，适度展开                        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  第 3 层：概念解释（深度理解）                                 │
│  • server/async.mdx - 异步处理机制                           │
│  • guides/about.mdx - Feat 概述                              │
│  职责：解释原理，帮助读者理解设计思想                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  第 4 层：API 参考（精确查阅）                                │
│  • cloud/options.mdx - CloudOptions 配置参考                 │
│  • server/serveroptions.mdx - ServerOptions 配置参考         │
│  职责：列举所有参数和方法，便于快速查阅                        │
└─────────────────────────────────────────────────────────────┘
```

### 引用而非重复原则

**禁止行为**：
- ❌ 在多个文档中重复介绍同一个概念
- ❌ 在每个文档开头都介绍环境配置
- ❌ 在操作指南中详细讲解基础用法

**正确做法**：
- ✅ 首次出现时详细解释，后续用链接引用
- ✅ 教程聚焦「完成一件事」，前置条件用链接指向入门文档
- ✅ 操作指南假设读者已完成入门，直接切入主题

### 交叉引用规范

```markdown
<!-- 引用入门文档 -->
开始之前，请确保已完成 [Feat Cloud 快速入门](/feat/cloud/getstart/)。

<!-- 引用概念文档 -->
关于异步处理的原理，请参考 [异步响应机制](/feat/server/async/)。

<!-- 引用参考文档 -->
完整的配置选项列表，请查看 [CloudOptions 配置参考](/feat/cloud/options/)。

<!-- 引用相关教程 -->
完成本教程后，可以继续学习 [MyBatis 集成](/feat/cloud/db/)。
```

### 内容边界判定表

| 当前文档类型 | 应该详写 | 应该链接 | 应该避免 |
|-------------|---------|---------|---------|
| **入门教程** | 环境搭建、第一个程序、验证运行 | 深入的功能教程、概念解释 | API 详细参数、高级配置 |
| **功能教程** | 完成特定功能的完整步骤 | 前置的入门教程、相关的概念解释 | 基础环境配置、无关的高级特性 |
| **操作指南** | 解决问题的具体步骤 | 涉及的 API 参考、相关教程 | 背景原理讲解、入门知识 |
| **概念解释** | 设计原理、架构分析、权衡比较 | 涉及的具体 API 参考 | 操作步骤、入门引导 |
| **参考文档** | 参数列表、方法签名、返回值 | 相关的使用教程 | 原理讲解、长篇示例 |

### 写作自检清单（体系化）

在完成每篇文档前，确认：

- [ ] **无重复内容**：本文档介绍的内容是否已在其他文档中出现？如果是，改为链接引用
- [ ] **前置条件明确**：是否需要先阅读其他文档？在开头给出链接
- [ ] **延伸路径清晰**：文档结尾是否有指向下一步学习的链接？
- [ ] **交叉引用有效**：文档内的所有链接都已验证可访问
- [ ] **层级定位准确**：本文档在整个文档体系中处于哪一层？是否符合该层的写作规范

## 质量检查清单

### 内容质量
- [ ] 文档类型明确且结构符合该类型的模板
- [ ] 标题层级正确（H1→H2→H3），无跳跃
- [ ] 含适当示例，步骤或操作清晰易懂
- [ ] 术语首次出现时有通俗解释
- [ ] 人称使用符合文档类型要求

### 技术准确性
- [ ] 所有代码示例可运行
- [ ] 代码来源于 `feat-test` 或经过实际验证
- [ ] 严格符合 JDK 8 规范
- [ ] Gitee 链接有效且指向正确

### 格式规范
- [ ] 代码块有语法高亮（java/xml/yaml/shell 等）
- [ ] Aside 组件使用恰当（tip/caution/danger）
- [ ] 表格对齐正确，信息完整
- [ ] Mermaid 图表有助于理解（如有）

### 用户体验
- [ ] 开篇吸引读者，结尾有下一步指引
- [ ] 步骤间有过渡语，逻辑流畅
- [ ] 关键信息突出（加粗、表格、代码）
- [ ] 无类型混淆（教程带人做、指南解任务、概念讲原理、参考列 API）

### 体系化
- [ ] 无内容重复，应链接的地方用了链接
- [ ] 前置条件明确，有相关文档链接
- [ ] 结尾有下一步学习建议
- [ ] 在整个文档体系中有明确定位
