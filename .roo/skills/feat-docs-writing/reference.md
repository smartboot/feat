## 四大文档类型详细模板

### 教程 (Tutorials)

**目标**：让读者从零开始完成一个完整的学习目标，建立信心。

**黄金法则**：
- 前 50% 内容内必须有可运行的代码示例
- 每完成一步都有明确的验证方式（能看到效果）
- 避免解释原理，专注「怎么做」

**推荐结构**：
```markdown
## [动词+目标，如：构建你的第一个 HTTP 服务器]

### 你将学到
- 学会 xxx
- 掌握 yyy
- 实现 zzz

### 前置要求
- JDK 8+
- Maven 3.6+
- （其他依赖）

### 步骤 1：xxxx
[简要说明这一步做什么]

```java title="文件名.java"
// 完整可运行代码
```

[验证方式：如何确认这一步成功]

### 步骤 2：xxxx
...

### 步骤 N：验证结果
[完整的测试方法，确保读者能确认成功]

### 总结回顾
- 我们完成了什么
- 关键概念回顾
- 下一步学习建议
```

**人称**：多用「你」指导，适当「我们」共学。亲切友好，像导师带学生。

### 操作指南 (How-to Guides)

**目标**：解决一个具体问题，提供多种方案供选择。

**黄金法则**：
- 问题描述 → 推荐方案 → 替代方案 → 注意事项
- 至少提供一种完整可运行的解决方案
- 包含常见错误和排错方法

**推荐结构**：
```markdown
## 如何 [具体任务]

### 问题描述
[什么场景下会遇到这个问题]

### 推荐方案
[最常用/最佳的解决方案]

```java
// 完整示例代码
```

**配置说明**：
| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| xxx | 是 | - | xxx |

### 替代方案
#### 方案 A：[名称]
[适用场景]：xxx
[优点]：xxx
[缺点]：xxx

#### 方案 B：[名称]
...

### 注意事项
- ⚠️ 常见问题 1
- ⚠️ 常见问题 2

### 相关链接
- [链接到相关教程]
- [链接到 API 参考]
```

**人称**：主要「你」，少用「我们」。专业直接，像技术支持。

### 概念解释 (Explanations)

**目标**：帮助读者理解「为什么这样设计」「底层原理是什么」。

**黄金法则**：
- 用类比解释抽象概念（如 NIO 类比餐厅服务）
- 结合架构图或流程图说明
- 分析设计权衡（为什么选 A 而不是 B）

**推荐结构**：
```markdown
## [主题] 解析 / 机制 / 原理

### 背景和动机
[为什么会出现这个概念/技术]
[解决了什么问题]

### 核心设计思想
[用最通俗的语言解释设计理念]
[适当的类比帮助理解]

### 实现原理
```mermaid
[架构图/流程图/时序图]
```

[关键代码片段说明原理，非完整示例]

### 性能优势与权衡
| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| A | ... | ... | ... |
| B | ... | ... | ... |

### 扩展思考
- 与其他技术的对比
- 演进历史
- 最佳实践建议
```

**人称**：多用「我们」引导思考，少用「你」。客观理性，像技术分享。

### 参考文档 (References)

**目标**：快速查阅 API、参数、返回值等精确信息。

**黄金法则**：
- 信息密度高，便于快速查找
- 每个参数都有类型、默认值、说明
- 简短准确的示例（不需要完整 main）

**推荐结构**：
```markdown
## [组件/API] 参考

### API 概览
[一句话描述这个组件的作用]

```java
// 典型的使用模式
Component c = new Component();
c.method();
```

### 构造方法

#### `ComponentName(Type param)`
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| param | Type | 是 | - | 参数说明 |

### 配置方法

#### `methodName(Type param)`
[方法作用简述]

**参数**：
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| param | Type | 否 | null | 参数说明 |

**返回值**：ReturnType - 返回值说明

**示例**：
```java
// 简短示例，展示用法即可
component.method(value);
```

### 异常处理
| 异常 | 触发条件 | 处理方式 |
|------|----------|----------|
| XxxException | 当...时抛出 | 应该... |

### 版本差异
| 版本 | 变更 |
|------|------|
| v1.4.0 | 新增 xxx 方法 |
| v1.3.0 | 废弃 yyy 方法 |
```

**人称**：第三人称，客观中立。简洁精确，像官方手册。

## 代码示例规范

### 示例来源优先级

1. **首选**：`feat-test` 模块中的真实可运行代码
2. **次选**：`feat-ai`、`demo/` 目录中的项目内示例
3. **补充**：为文档专门编写的精简示例（但必须可运行）

### 代码注释模板

根据文档类型使用不同的注释头：

**教程**：
```java
/**

 * 
 * @see feat-test/src/main/java/tech/smartboot/feat/demo/HelloWorld.java
 */
```

```java
