# 质量检查清单

> **核心目标**：确保文档符合 Feat 规范，可被 AI 准确理解和执行。

---

## 发布前检查

### 代码示例检查

- [ ] 代码来自 `demo/` 或 `feat-test/`，可运行
- [ ] 代码 JDK 8 兼容（无 `var`、`record`、文本块等）
- [ ] 包含完整依赖（pom.xml 片段）
- [ ] 每个代码块后有验证步骤（编译→运行→测试→预期输出）
- [ ] 包含常见错误及解决方案
- [ ] 代码模式已标注（Pattern-XX）
- [ ] 代码包含文件路径标注

### 文档结构检查

- [ ] 使用标准 Markdown（标题层级 H1→H2→H3，不跳级）
- [ ] 代码块指定语言（`java`、`xml`、`bash` 等）
- [ ] 表格格式正确（表头对齐）
- [ ] 内部链接有效（相对路径正确）
- [ ] 文档包含前置知识说明
- [ ] 文档包含下一步指引

### 内容完整性检查

- [ ] 每个概念包含：定义、使用场景、代码示例
- [ ] API 说明包含参数类型、返回值
- [ ] 版本信息明确（如 Feat ≥ 3.2.0）
- [ ] 术语使用一致
- [ ] 反模式包含 ❌/✅ 对比示例

### AI 可解析性检查

- [ ] 使用结构化格式（表格、列表）而非纯文本
- [ ] 代码示例包含完整的上下文（包、导入、类）
- [ ] 验证步骤包含具体命令和预期输出
- [ ] 错误诊断包含现象、原因、解决三步
- [ ] 文档间引用使用相对路径
- [ ] 代码模式引用使用标准格式 `[Pattern-XX](05-code-patterns.md)`

## 自动化检查脚本

### 检查脚本 1：代码完整性

```bash
#!/bin/bash
# check-code.sh - 检查代码示例完整性

echo "=== 代码完整性检查 ==="

# 检查代码块是否包含包声明
grep -n "^package " "$1" | wc -l
echo "✓ 包含包声明的代码块数"

# 检查是否包含导入语句
grep -n "^import " "$1" | wc -l
echo "✓ 包含导入语句的代码块数"

# 检查是否包含验证步骤
grep -n "**验证步骤**" "$1" | wc -l
echo "✓ 包含验证步骤的代码块数"

# 检查是否标注代码模式
grep -n "Pattern-" "$1" | wc -l
echo "✓ 引用代码模式的次数"
```

### 检查脚本 2：术语一致性

```bash
#!/bin/bash
# check-terminology.sh - 检查术语一致性

echo "=== 术语一致性检查 ==="

# 定义标准术语
TERMS=("Router" "Context" "Interceptor" "Handler" "FeatAI" "ChatModel")

for term in "${TERMS[@]}"; do
    count=$(grep -o "$term" "$1" | wc -l)
    echo "$term: 出现 $count 次"
done

# 检查非标准术语
echo ""
echo "=== 可能的非标准术语 ==="
grep -n "路由组件\|请求上下文\|拦截器" "$1" || echo "未发现明显非标准术语"
```

### 检查脚本 3：链接有效性

```bash
#!/bin/bash
# check-links.sh - 检查内部链接有效性

echo "=== 链接有效性检查 ==="

# 提取所有 Markdown 链接
links=$(grep -oE '\[([^]]+)\]\(([^)]+)\)' "$1" | grep -v "http" | sed 's/.*(\(.*\))/\1/')

for link in $links; do
    # 移除 # 锚点
    clean_link="${link%%#*}"
    
    if [ -f "$clean_link" ] || [ -f "$clean_link.md" ] || [ -f "$clean_link.mdx" ]; then
        echo "✓ $link"
    else
        echo "✗ $link (文件不存在)"
    fi
done
```

## 质量评分标准

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 代码完整性 | 30% | 代码是否完整、可运行 |
| 文档结构 | 20% | 标题层级、格式规范 |
| 内容准确性 | 25% | 技术内容是否正确 |
| AI 可解析性 | 15% | 结构化程度、机器可读性 |
| 用户体验 | 10% | 易读性、导航清晰度 |

### 评分等级

| 等级 | 分数 | 说明 |
|------|------|------|
| 优秀 | 90-100 | 可直接发布 |
| 良好 | 80-89 | 需小修改 |
| 合格 | 70-79 | 需较大修改 |
| 不合格 | <70 | 需重写 |

## 常见问题快速修复

### 问题 1：代码不完整

**快速修复**：
1. 添加包声明：`package com.example;`
2. 添加必要导入
3. 添加类注释（功能、版本）
4. 添加 main 方法
5. 添加验证步骤

### 问题 2：缺少验证步骤

**快速修复模板**：

```mdx
**验证步骤**：
1. 编译：`mvn compile`
2. 运行：`mvn exec:java -Dexec.mainClass="..."`
3. 测试：`curl ...`
4. 预期输出：`...`
```

### 问题 3：术语不一致

**快速修复**：
1. 建立术语表
2. 统一替换非标准术语
3. 首次出现时添加定义

### 问题 4：链接失效

**快速修复**：
1. 使用相对路径：`/feat/cloud/controller/`
2. 避免绝对路径：`https://...`
3. 定期检查链接有效性
