# 质量检查清单

> **核心目标**：确保文档符合 Feat 规范，可被 AI 准确理解和执行。

---

## 发布前检查

### 代码示例检查

- [ ] 代码来自 `demo/` 或 `feat-test/`，可运行
- [ ] **代码自包含**：AI 可以直接从文档获取完整代码，无需访问外部文件
- [ ] 代码 JDK 8 兼容（无 `var`、`record`、文本块等）
- [ ] 包含完整依赖（pom.xml 片段）
- [ ] 每个代码块后有验证步骤（编译→运行→测试→预期输出）
- [ ] 代码包含文件路径标注
- [ ] 不依赖外部代码链接（如"完整代码参见 xxx"）

### 文档结构检查

- [ ] 使用标准 Markdown（标题层级 H1→H2→H3，不跳级）
- [ ] 代码块指定语言（`java`、`xml`、`bash` 等）
- [ ] 表格格式正确（表头对齐）
- [ ] 内部链接有效（相对路径正确）
- [ ] 文档包含前置知识说明
- [ ] 文档包含下一步指引

### 内容完整性检查

- [ ] 每个概念包含：定义、使用场景、代码示例
- [ ] API 说明包含参数类型、返回值
- [ ] 版本信息明确（如 Feat ≥ 2.0.0）
- [ ] 术语使用一致
- [ ] 反模式包含 ❌/✅ 对比示例

### AI 可解析性检查

- [ ] 使用结构化格式（表格、列表）而非纯文本
- [ ] 代码示例包含完整的上下文（包、导入、类）
- [ ] **代码自包含**：AI 无需访问外部文件即可获取完整代码
- [ ] 验证步骤包含具体命令和预期输出
- [ ] 错误诊断包含现象、原因、解决三步
- [ ] 文档间引用使用相对路径
- [ ] 无"完整代码参见 xxx"之类的外部依赖

---

## 常见问题快速修复

### 问题 1：代码不完整

**快速修复**：
1. 添加包声明：`package com.example;`
2. 添加必要导入
3. 添加类注释（功能、版本）
4. 添加 main 方法
5. 添加验证步骤

### 问题 2：缺少验证步骤

**快速修复模板**：

```mdx
**验证步骤**：
1. 编译：`mvn compile`
2. 运行：`mvn exec:java -Dexec.mainClass="..."`
3. 测试：`curl ...`
4. 预期输出：`...`
```

### 问题 3：代码不自包含

**快速修复**：
1. 添加包声明：`package com.example;`
2. 添加所有必要导入语句
3. 添加完整的类定义
4. 添加 main 方法
5. 移除"完整代码参见 xxx"之类的外部依赖
6. 确保代码可以直接复制使用

### 问题 4：代码示例不存在

**快速修复**：
1. **停止文档编写**
2. 在 `demo/` 或 `feat-test/` 中创建完整的可运行示例
3. 确保示例代码包含包声明、导入、类定义、main 方法
4. 编译并运行示例代码，确保其可正常工作
5. 基于已验证的示例代码继续编写文档
