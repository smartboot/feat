# 第 1 章：项目启动

## 本章目标

完成本章后，你将能够：
- ✅ 创建一个 Feat Cloud 项目
- ✅ 实现第一个 HTTP 端点
- ✅ 理解 Feat 的基本项目结构
- ✅ 运行并测试你的应用

## 欢迎来到 Taco Cloud

想象一下，你接到了一个新项目：为一家玉米卷店开发在线订购系统。
顾客可以浏览菜单、自定义玉米卷、下单购买。

这就是 **Taco Cloud**——我们的实战项目。

在本章中，我们将从最基础开始：创建一个 Feat 项目，并实现一个欢迎页面。

## 创建项目

### 步骤 1：创建 Maven 项目

首先，创建一个标准的 Maven 项目结构：

```bash
mkdir taco-cloud
cd taco-cloud
mkdir -p src/main/java/com/tacocloud
```

### 步骤 2：添加 Feat 依赖

创建 `pom.xml` 文件：

```xml title="pom.xml"
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.tacocloud</groupId>
    <artifactId>taco-cloud</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <properties>
        <maven.compiler.source>1.8</maven.compiler.source>
        <maven.compiler.target>1.8</maven.compiler.target>
    </properties>

    <dependencies>
        <dependency>
            <groupId>tech.smartboot.feat</groupId>
            <artifactId>feat-cloud</artifactId>
            <version>3.2.0</version>
        </dependency>
    </dependencies>
</project>
```

> 💭 **你可能想知道**：为什么是 `feat-cloud` 而不是 `feat-core`？
> 
> `feat-cloud` 包含了依赖注入、配置管理等高级特性，适合构建完整的应用。
> 如果只是简单的 HTTP 服务，`feat-core` 就够了。

### 步骤 3：创建启动类

创建 `TacoCloudApplication.java`：

```java title="src/main/java/com/tacocloud/TacoCloudApplication.java"
package com.tacocloud;

import tech.smartboot.feat.cloud.FeatCloud;
import tech.smartboot.feat.cloud.CloudService;

public class TacoCloudApplication {
    public static void main(String[] args) {
        CloudService server = FeatCloud.cloudServer();
        
        // 首页端点
        server.get("/", ctx -> {
            ctx.write("欢迎来到 Taco Cloud！");
        });
        
        server.listen();
    }
}
```

## 运行应用

### 编译并运行

```bash
mvn compile exec:java -Dexec.mainClass="com.tacocloud.TacoCloudApplication"
```

你应该看到类似输出：

```
[INFO] --- exec-maven-plugin:3.1.0:java (default-cli) @ taco-cloud ---
Feat Cloud Server started on port 8080
```

### 测试

打开浏览器，访问 http://localhost:8080

你应该看到：

```
欢迎来到 Taco Cloud！
```

🎉 **恭喜！** 你的第一个 Taco Cloud 端点已经运行起来了。

## 深入理解

### 刚才发生了什么？

让我们逐行分析刚才的代码：

```java
CloudService server = FeatCloud.cloudServer();
```

这行代码创建了一个 Feat Cloud 服务实例。`FeatCloud` 是入口类，`cloudServer()` 方法返回一个配置好的服务。

```java
server.get("/", ctx -> {
    ctx.write("欢迎来到 Taco Cloud！");
});
```

这里我们定义了一个路由：
- `get("/")`：处理 GET 请求，路径是根路径 `/`
- `ctx -> { ... }`：Lambda 表达式，处理请求的逻辑
- `ctx.write(...)`：向客户端写入响应内容

```java
server.listen();
```

启动服务器，默认监听 8080 端口。

### 常见错误：忘记调用 listen()

**试试这段代码**：

```java
public static void main(String[] args) {
    CloudService server = FeatCloud.cloudServer();
    server.get("/", ctx -> ctx.write("Hello"));
    // 忘记调用 listen()
}
```

**你会看到**：程序立即退出，没有任何输出。

**为什么**：`listen()` 是阻塞方法，启动服务器并等待请求。如果不调用，程序会直接结束。

## 动手实验：探索路由

### 实验 1：添加更多端点

修改代码，添加以下端点：

```java
server.get("/menu", ctx -> {
    ctx.write("玉米卷菜单");
});

server.get("/order", ctx -> {
    ctx.write("订单页面");
});
```

**测试**：
- http://localhost:8080/menu
- http://localhost:8080/order

### 实验 2：观察 404

访问一个不存在的路径：http://localhost:8080/unknown

**你会看到**：404 页面

**思考**：如何自定义 404 页面？（我们将在后续章节学习）

## 项目结构

现在，你的项目结构应该是：

```
taco-cloud/
├── pom.xml
└── src/
    └── main/
        └── java/
            └── com/
                └── tacocloud/
                    └── TacoCloudApplication.java
```

这是一个标准的 Maven 项目结构。随着项目发展，我们会添加更多包和类。
