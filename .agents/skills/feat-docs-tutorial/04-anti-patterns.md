# 反模式目录

以下写作方式必须避免。

## 反模式0：机械套用模板（新增）

```mdx
// ❌ 错误：每篇文章结构雷同，缺乏个性

本文介绍 xxx 功能...

## 基础用法
### 最简示例
```java
// 代码
```

### 参数说明
| 参数 | 类型 | 说明 |

## 进阶用法
### 场景1
...

### 场景2
...

## 最佳实践
...

## 常见问题
...

// ✅ 正确：根据内容特点选择叙事结构
## 场景：为什么需要 xxx
假设你正在开发...

## 挑战：传统方案的问题
传统做法会遇到...

## 解决：Feat 的优雅方案
Feat 通过...解决了这个问题

## 实现：完整代码示例
```java
// 代码
```

## 延伸：还可以做什么
这个方案还可以...
```

**问题诊断：**
- 读者感觉在读说明书，而非跟一个有经验的开发者对话
- 每篇文章结构雷同，缺乏记忆点
- 内容为了填满结构而写，而非为了解决问题

**解决策略：**
1. 动笔前先确定"故事"：这篇文章要解决什么具体问题？
2. 选择叙事结构：PES（问题-探索-解决）/ CEP（概念-演进-实践）/ SSI（场景-选型-实现）
3. 用场景开头："假设你正在开发..."
4. 展示思考过程："一开始我想到的是..."
5. 章节标题具体化：将"功能概述"改为"为什么需要 xxx"
