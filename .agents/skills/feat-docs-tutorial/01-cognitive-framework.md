# 认知目标框架

在动笔前，先明确本教程属于哪种认知目标类型，这将决定内容结构和深度。

## 认知目标分类表

| 认知目标 | 读者状态 | 内容策略 | 典型结构 |
|---------|---------|---------|---------|
| **记忆** | 初次接触 | 重复、关联、助记 | 定义 + 示例 + 速查表 |
| **理解** | 听说过但不懂原理 | 类比、可视化、多角度解释 | 概念 → 原理 → 示例 |
| **应用** | 理解原理但不会用 | 步骤化、脚手架、常见场景 | 环境 → 步骤 → 验证 |
| **分析** | 会用但不知为何这样 | 对比、拆解、边界情况 | 正确 vs 错误对比 |
| **评估** | 能独立使用 | 最佳实践、性能对比、选型指南 | 场景 → 方案 → 对比 |
| **创造** | 熟练掌握 | 架构模式、扩展点、集成方案 | 需求 → 设计 → 实现 |

## 决策规则

| 教程类型 | 认知目标 | 字数控制 |
|---------|---------|---------|
| 快速入门 | 应用（30 分钟内跑通第一个 Demo） | 800-1500 字 |
| 功能教程 | 应用 → 分析（会用 + 懂边界情况） | 1500-3000 字 |
| 集成教程 | 应用（跑通集成）→ 评估（选型依据） | 2000-4000 字 |
| 最佳实践 | 评估 → 创造（选型 + 架构） | 1500-2500 字 |

## 认知负荷管理

- 每个代码块不超过 20 行
- 每个章节不超过 3 个代码块
- 复杂示例必须拆解为多个步骤

### 心智模型构建

**每个新概念首次出现时，必须回答三个问题：**
1. **What** - 这是什么？（一句话定义）
2. **Why** - 为什么需要？（解决什么问题）
3. **How** - 怎么用？（最小代码示例）

**示例：**

```mdx
## Router 路由组件

**What：** Router 负责将 HTTP 请求分发到对应的处理器。

**Why：** 没有 Router，所有请求都走同一个入口，代码会迅速膨胀为"面条代码"。

**How：**
\```java
router.get("/users", ctx -> ctx.write("用户列表"));
router.post("/users", ctx -> ctx.write("创建用户"));
\```
```

### 视觉层次设计

**使用标题层级控制阅读节奏：**

| 层级 | 用途 | 每篇文档数量 |
|------|------|-------------|
| H1 (`#`) | 文档标题 | 1 |
| H2 (`##`) | 主要章节 | 3-5 |
| H3 (`###`) | 子主题 | 每个 H2 下 2-4 |
| H4 (`####`) | 细分点 | 尽量避免，用列表替代 |

**关键规则：**
- 章节标题必须包含动作或结果，避免纯名词
  - ❌ `配置` → ✅ `配置数据库连接`
- 同级标题之间保持逻辑平行
- 不要跳级（H2 下直接 H4）
