# 写作哲学：从模板到叙事

## 当前问题诊断

### ❌ 模板化陷阱

```
问题表现：
1. 每篇文章结构雷同：概述 → 基础用法 → 进阶用法 → 最佳实践 → 常见问题
2. 章节标题机械："功能概述"、"基础用法"、"进阶用法"
3. 内容填充感强：为了填满结构而写，而非为了解决问题
4. 缺乏情感连接：读者感觉在读说明书，而非跟一个有经验的开发者对话
```

### ❌ 连贯性缺失

```
问题表现：
1. 单篇文档独立，与其他文档无关联
2. 知识点之间没有递进关系
3. 读者学完 A 后，不知道为什么要学 B
4. 缺乏学习路径的引导
```

## 新写作哲学：叙事驱动

### 核心转变

| 维度 | 旧模式 | 新模式 |
|------|--------|--------|
| **驱动力** | 结构模板 | 用户故事 |
| **组织方式** | 功能分类 | 学习路径 |
| **语气风格** | 说明文档 | 对话交流 |
| **内容来源** | 代码示例 | 真实场景 |
| **目标导向** | 覆盖功能 | 解决问题 |

### 叙事三要素

每篇文档必须回答：

1. **Who** - 谁在什么场景下遇到这个问题？
2. **What** - 他们想要达成什么目标？
3. **Why** - 为什么选择 Feat 的解决方案？

## 叙事结构模板

### 模板1：问题-探索-解决（PES）

适用于：解决具体技术问题的教程

```
1. 场景引入（故事化）
   "假设你正在开发一个电商系统，需要处理用户下单..."

2. 遇到的挑战（产生共鸣）
   "传统的做法会遇到以下问题..."

3. 探索方案（对比分析）
   "我们尝试过 A、B、C 三种方案..."

4. Feat 的解决方式（核心内容）
   "Feat 通过 xxx 特性优雅地解决了这个问题..."

5. 完整实现（可运行代码）
   "下面是完整的实现代码..."

6. 效果验证（可验证）
   "运行后，你可以看到..."

7. 延伸思考（启发性）
   "这个方案还可以扩展到..."
```

### 模板2：概念-演进-实践（CEP）

适用于：需要理解设计思想的教程

```
1. 核心概念（一句话定义）
   "依赖注入的本质是..."

2. 演进历程（为什么需要）
   "在没有 DI 之前，我们是这样做的..."
   "这种方式的问题在于..."
   "于是产生了 DI 模式..."

3. Feat 的实现（具体用法）
   "Feat 的 DI 设计遵循以下原则..."

4. 实践案例（真实场景）
   "来看一个用户服务的实际案例..."

5. 常见误区（避坑指南）
   "初学者容易犯这些错误..."
```

### 模板3：场景-选型-实现（SSI）

适用于：多种方案对比的教程

```
1. 业务场景（具体描述）
   "你需要为一个高并发系统选择缓存方案..."

2. 方案对比（客观分析）
   | 方案 | 适用场景 | 优缺点 |

3. Feat 支持（能力展示）
   "Feat 支持以上所有方案，使用方式分别是..."

4. 推荐选择（给出建议）
   "对于你的场景，我建议..."

5. 实现步骤（详细指导）
   "下面是具体的配置步骤..."
```

## 个性化写作指南

### 根据内容特点选择结构

| 内容类型 | 推荐结构 | 语气风格 |
|----------|----------|----------|
| 快速入门 | 步骤式 | 鼓励、简洁 |
| 概念讲解 | 演进式 | 启发、类比 |
| 功能教程 | 场景式 | 实用、直接 |
| 最佳实践 | 案例式 | 经验分享 |
| 故障排查 | 问题-解决 | 共情、清晰 |
| 架构设计 | 决策式 | 理性、全面 |

### 避免模板化的技巧

1. **用故事开头**："假设你正在开发..."
2. **用对话语气**："你可能会问..."、"让我们来看看..."
3. **展示思考过程**："一开始我想到的是..."
4. **承认复杂性**："这部分确实有点复杂..."
5. **给出选择理由**："之所以这样设计，是因为..."

## 质量检查：连贯性专项

### 单篇检查

- [ ] 开头是否建立了场景/问题？
- [ ] 内容是否围绕核心问题展开？
- [ ] 是否有明确的"为什么"解释？
- [ ] 结尾是否给出下一步指引？

### 系列检查

- [ ] 系列文档是否有统一的学习路径？
- [ ] 相邻文档之间是否有承上启下？
- [ ] 知识点是否按递进关系组织？
- [ ] 是否有重复内容（应改为链接）？

### 全站检查

- [ ] 相同概念在不同文档中表述一致？
- [ ] 代码示例风格统一？
- [ ] 术语使用一致？
- [ ] 难度梯度合理？
