# Feat 图片获取与管理方案

## 1. 方案概述

为确保 Feat 版本发布文档和微信公众号文章的图片质量和稳定性，本方案提供了多种图片获取渠道和管理策略，避免对单一不稳定 API 的依赖。

## 2. 图片获取渠道

### 2.1 官方资源库

#### 品牌素材库
- **建立方式**：与设计团队合作，创建 Feat 品牌素材库
- **内容包括**：logo、图标、配色方案、背景图等
- **更新频率**：每季度更新一次，版本发布前补充
- **存储位置**：`feat/images/common/` 目录

#### 版本专用图片
- **创建时机**：每个版本发布前 2 周
- **内容包括**：版本封面图、功能示意图、使用场景图
- **存储位置**：`feat/images/v{版本号}/` 目录
- **命名规范**：`feat-{版本号}-{图片类型}.{格式}`

### 2.2 设计团队协作

#### 流程
1. **需求提交**：文档编辑员提前 2 周向设计团队提交图片需求
2. **设计制作**：设计团队根据需求制作符合品牌风格的图片
3. **审核确认**：文档编辑员审核图片质量和风格
4. **入库存储**：将确认后的图片存储到对应目录

#### 工具推荐
- **设计工具**：Figma、Sketch、Adobe Illustrator
- **协作工具**：Trello、Jira、企业微信

### 2.3 免费图库资源

#### 推荐图库
- **Unsplash**：https://unsplash.com/ - 高质量免费图片，无需 attribution
- **Pexels**：https://www.pexels.com/ - 免费图片和视频资源
- **Pixabay**：https://pixabay.com/ - 免费图片、矢量图和视频
- **Freepik**：https://www.freepik.com/ - 提供免费和付费资源

#### 使用准则
- **版权检查**：确保图片可用于商业用途
- **风格匹配**：选择与 Feat 品牌风格一致的图片
- **分辨率要求**：至少 1920×1080 像素
- **后期处理**：根据需要调整色彩和风格

### 2.4 截图资源

#### 产品截图
- **界面截图**：使用 Feat 实际运行界面的截图
- **功能演示**：截图展示功能使用流程
- **对比截图**：展示功能改进前后的对比

#### 代码可视化
- **Carbon**：https://carbon.now.sh/ - 生成美观的代码截图
- **CodeSnap**：VS Code 插件，生成代码截图
- **Polacode**：VS Code 插件，生成代码截图

#### 截图工具
- **Snagit**：专业截图工具，支持编辑和标注
- **Snipping Tool**：Windows 内置截图工具
- **Grab**：Mac 内置截图工具
- **Lightshot**：跨平台截图工具

### 2.5 应急图片获取

#### 快速生成
- **Canva**：https://www.canva.com/ - 在线设计工具，提供模板
- **Figma**：https://www.figma.com/ - 协作设计工具
- **PowerPoint**：快速创建简单的示意图

#### 备用资源
- **图标库**：Font Awesome、Material Icons
- **插图库**：Noun Project、Flaticon
- **应急联系人**：设计团队紧急联系方式

## 3. 图片管理策略

### 3.1 目录结构

```
feat/
└── images/
    ├── common/            # 通用品牌素材
    │   ├── logo/          # logo 相关图片
    │   ├── icons/         # 图标
    │   └── backgrounds/   # 背景图
    ├── v1.5.0/            # 版本专用图片
    │   ├── cover/         # 封面图
    │   ├── features/      # 功能示意图
    │   └── screenshots/   # 截图
    ├── v1.5.1/            # 其他版本
    └── temp/              # 临时图片
```

### 3.2 命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 品牌logo | `feat-logo-{变体}.{格式}` | `feat-logo-primary.png` |
| 版本封面 | `feat-{版本号}-cover.{格式}` | `feat-1.5.0-cover.png` |
| 功能示意图 | `feat-{版本号}-{功能名}.{格式}` | `feat-1.5.0-ai-agent.png` |
| 截图 | `feat-{版本号}-screenshot-{序号}.{格式}` | `feat-1.5.0-screenshot-1.png` |
| 代码图 | `feat-{版本号}-code-{功能名}.{格式}` | `feat-1.5.0-code-reactor.png` |

### 3.3 存储管理

#### 本地存储
- **版本控制**：将图片纳入 Git 版本控制系统
- **压缩优化**：使用图片压缩工具减少存储和加载成本
- **备份策略**：定期备份到云存储

#### 云存储
- **GitHub Pages**：使用 GitHub 仓库存储图片
- **云服务**：使用阿里云OSS、腾讯云COS等服务
- **CDN加速**：配置 CDN 加速图片访问

### 3.4 质量控制

#### 图片质量标准
- **分辨率**：封面图至少 900×383 像素，内容图至少 800×600 像素
- **文件大小**：封面图 < 500KB，内容图 < 300KB
- **格式**：优先使用 PNG 格式，其次 JPG
- **色彩**：符合 Feat 品牌色彩方案

#### 审核流程
1. **初步审核**：设计团队内部审核
2. **内容审核**：文档编辑员审核内容准确性
3. **质量审核**：检查分辨率、大小、格式等
4. **最终确认**：确认图片符合所有标准

## 4. 图片使用指南

### 4.1 Markdown 插入方法

#### 基本语法

```markdown
![图片描述](图片路径)
```

#### 本地图片引用

```markdown
![Feat 框架 logo](../images/common/logo/feat-logo-primary.png)
![AI Agent 功能](../images/v1.5.0/features/feat-1.5.0-ai-agent.png)
```

#### 网络图片引用

```markdown
![Feat 框架 logo](https://example.com/feat-logo.png)
```

#### 高级用法

**指定尺寸**：

```markdown
<img src="图片路径" width="600" height="400" alt="图片描述">
```

**图片居中**：

```markdown
<div align="center">
  <img src="图片路径" alt="图片描述">
</div>
```

**图片带说明**：

```markdown
![图片描述](图片路径)
*图片说明文字*
```

### 4.2 微信公众号使用

#### 封面图
- **尺寸**：900×383 像素
- **格式**：JPG 或 PNG
- **大小**：< 500KB
- **要求**：包含版本号和核心亮点，与文章主题高度相关
- **设计建议**：使用品牌色系，突出核心功能，添加简洁的视觉元素

#### 内容图片
- **尺寸**：宽度 800-1000 像素
- **格式**：JPG 或 PNG
- **大小**：< 300KB
- **要求**：清晰、相关、符合品牌风格
- **内容匹配度**：图片内容应与相邻文字内容直接相关，能够直观传达文章核心信息

#### 图片布局
- **位置**：放在相关内容附近，保持上下文连贯
- **间距**：图片与文字之间保持适当间距，建议 1-2 行
- **对齐**：使用微信公众号的居中功能，确保图片居中显示
- **分组**：相关图片可以分组展示，使用统一的布局风格

#### 图片说明
- **长度**：简短明了，1-2 句话即可
- **内容**：说明图片的核心信息或价值
- **位置**：放在图片下方，使用斜体或较小字号

#### 插入方法
1. 在微信公众号后台上传图片
2. 使用微信编辑器的图片插入功能
3. 调整图片大小和位置
4. 添加图片说明（如有需要）
5. 检查图片在不同设备上的显示效果

## 5. 工作流程

### 5.1 版本发布图片准备流程

1. **需求分析**（发布前 4 周）
   - 分析版本发布内容
   - 确定需要的图片类型和数量
   - 制定图片需求清单

2. **资源准备**（发布前 3 周）
   - 从品牌素材库获取现有资源
   - 向设计团队提交新图片需求
   - 收集免费图库资源作为备选

3. **图片制作**（发布前 2 周）
   - 设计团队制作新图片
   - 文档编辑员准备截图和代码图
   - 审核所有图片质量

4. **图片入库**（发布前 1 周）
   - 将所有图片存储到对应目录
   - 确认命名规范和文件格式
   - 更新图片清单

5. **文档整合**（发布前 3 天）
   - 在文档中插入图片
   - 检查图片显示效果
   - 最终审核所有图片

6. **发布与归档**（发布后 1 周）
   - 发布文档和公众号文章
   - 收集用户反馈
   - 归档图片资源

### 5.2 应急处理流程

1. **图片缺失**
   - 检查品牌素材库是否有替代资源
   - 使用免费图库资源作为临时替代
   - 联系设计团队紧急制作

2. **图片质量问题**
   - 检查原始图片是否可用
   - 使用图片编辑工具修复
   - 重新获取或制作

3. **图片版权问题**
   - 立即替换为有版权的图片
   - 从品牌素材库或免费图库获取替代
   - 更新图片使用记录
