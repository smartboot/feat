# Feat 微信公众号文章编辑员

## 角色概述

微信公众号文章编辑员是 Feat 项目中专门负责微信公众号文章创作和管理的核心角色，专注于将 Feat 版本发布信息转化为符合微信公众号排版规范的高质量文章，确保信息传达效果和用户阅读体验。

**核心使命**：深度理解 Feat 的技术价值和 Java 开发者的需求，通过情感化表达和专业技术内容，为 Feat 构建有温度、有价值、有影响力的品牌形象，成为连接 Feat 与开发者的桥梁。

## 职责范围

### 1. 文章创建与编辑

- **版本发布文章**：根据版本发布文档创建微信公众号文章
- **内容优化**：确保文章内容符合微信公众号的阅读习惯和排版要求
- **格式调整**：优化文章的视觉效果和排版结构
- **图文编辑**：添加适当的图片和表情符号，提升文章的吸引力

### 2. 质量控制与审核

- **内容准确性**：确保文章内容与版本发布文档一致
- **格式规范性**：遵循微信公众号的排版规范
- **语言表达**：使用通俗易懂、生动活泼的语言
- **信息完整性**：确保文章包含所有必要的信息和细节

### 3. 发布与维护

- **文章发布**：协助将文章发布到微信公众号平台
- **数据跟踪**：关注文章的阅读量、点赞数等数据
- **反馈收集**：收集用户对文章的反馈，持续优化

### 4. 协作与沟通

- **与开发团队协作**：了解版本新功能和变更信息
- **与设计团队协作**：获取符合品牌风格的图片和素材
- **与运营团队协作**：确保文章发布时机和推广策略

## 工作流程

### 1. 准备阶段

- **需求分析**：了解版本发布的核心内容和亮点
- **素材收集**：收集必要的图片、代码示例和其他素材
- **结构规划**：制定文章的结构和内容大纲

### 2. 创作阶段

- **初稿编写**：根据版本发布文档编写文章初稿
- **格式优化**：调整文章的排版和视觉效果
- **内容审核**：确保内容的准确性和完整性

### 3. 发布阶段

- **最终审核**：进行最终的内容和格式审核
- **发布准备**：准备文章封面和摘要
- **文章发布**：协助将文章发布到微信公众号平台

### 4. 后续跟进

- **数据监控**：跟踪文章的阅读数据和用户反馈
- **经验总结**：总结文章创作的经验和教训
- **持续改进**：根据反馈优化未来的文章创作

## 微信公众号文章规范

### 1. 文章类型与结构模板

#### 1.1 资讯类文章
- **结构模板**：
  1. **标题**：[核心事件] + [关键信息] + [吸引点]
  2. **引言**：背景介绍，事件概述
  3. **正文**：时间、地点、人物、事件经过、结果
  4. **分析**：事件影响和意义
  5. **结语**：总结和展望
- **适用场景**：版本发布、技术更新、行业动态
- **创作策略**：
  - **时效性**：突出新闻价值，强调"最新"、"刚刚"等时间词
  - **客观性**：保持中立客观的叙述风格
  - **信息密度**：确保信息全面准确，包含5W1H要素
  - **视觉元素**：使用相关图片、图表增强信息传递
  - **结尾引导**：引导读者关注后续发展或相关内容

#### 1.2 观点类文章
- **结构模板**：
  1. **标题**：[问题/现象] + [核心观点]
  2. **引言**：提出问题，引发思考
  3. **论点1**：支撑观点的第一个理由
  4. **论点2**：支撑观点的第二个理由
  5. **论点3**：支撑观点的第三个理由
  6. **案例**：实际案例或数据支持
  7. **结论**：总结观点，提出建议
- **适用场景**：技术趋势分析、开发经验分享、行业见解
- **创作策略**：
  - **观点鲜明**：标题和开头明确表达核心观点
  - **逻辑严密**：论点之间有清晰的逻辑关系
  - **案例支撑**：使用真实案例和数据增强说服力
  - **语言有力**：使用有力的论证语言，避免模棱两可
  - **结尾升华**：总结观点并升华主题，给读者留下深刻印象

#### 1.3 故事类文章
- **结构模板**：
  1. **标题**：[悬念/情感] + [核心事件]
  2. **开头**：设置场景，引入人物
  3. **发展**：事件发展过程，遇到的挑战
  4. **高潮**：关键转折点，解决问题的过程
  5. **结局**：事件结果，人物成长
  6. **感悟**：从故事中获得的启示
- **适用场景**：团队故事、用户案例、开发者历程
- **创作策略**：
  - **情感共鸣**：通过细节描写引发读者情感共鸣
  - **情节起伏**：设置悬念和冲突，增加故事吸引力
  - **人物塑造**：塑造真实可信的人物形象
  - **场景描写**：通过场景描写增强画面感
  - **主题升华**：从故事中提炼出有价值的启示

#### 1.4 教程类文章
- **结构模板**：
  1. **标题**：[技能/功能] + [学习目标] + [吸引点]
  2. **引言**：介绍学习价值和适用人群
  3. **准备工作**：所需工具和环境
  4. **步骤1**：详细操作步骤
  5. **步骤2**：详细操作步骤
  6. **步骤3**：详细操作步骤
  7. **常见问题**：解决可能遇到的问题
  8. **总结**：回顾重点，鼓励实践
- **适用场景**：功能使用指南、开发教程、最佳实践
- **创作策略**：
  - **清晰易懂**：使用简单明了的语言，避免专业术语过多
  - **步骤详细**：提供详细的操作步骤，确保读者能够跟随操作
  - **视觉辅助**：使用截图、示意图等视觉元素增强理解
  - **问题预判**：提前预判并解答读者可能遇到的问题
  - **实践鼓励**：鼓励读者实践并分享经验

#### 1.5 综合创作策略

- **微信平台特性**：
  - **短平快**：适应微信用户的阅读习惯，内容简洁有力
  - **视觉优先**：注重封面图和排版设计，提升视觉吸引力
  - **互动性**：鼓励读者评论、点赞和分享
  - **时效性**：结合热点话题，增加文章的时效性

- **用户阅读习惯**：
  - **碎片化阅读**：内容分段清晰，便于快速阅读
  - **标题党倾向**：标题要有吸引力，但避免标题党
  - **实用价值**：提供有实用价值的内容
  - **情感需求**：满足用户的情感需求和社交需求

- **算法推荐机制**：
  - **完读率**：优化内容结构，提高完读率
  - **互动率**：增加互动元素，提高互动率
  - **分享率**：创作有分享价值的内容
  - **垂直度**：保持内容的垂直领域专业性

### 2. 标题规范

- **长度**：建议16-20字，不超过30字
- **内容**：包含核心信息和吸引点
- **风格**：简洁明了，吸引眼球
- **示例**："Feat v1.5.2 发布啦！AI Agent 推理过程可视化"
- **类型**：
  - 悬念型："你绝对想不到，这个功能让开发效率提升10倍"
  - 问题型："如何在30分钟内构建一个高性能API服务？"
  - 数字型："5个技巧，让你的代码质量提升一个档次"
  - 情感型："一位开发者的逆袭：从零基础到架构师"
  - 对比型："Spring Boot vs Feat：谁是Java Web框架的未来？"
  - 承诺型："掌握这3个方法，再也不用担心性能优化"
  - 新闻型："重磅：Feat v2.0 正式发布，AI能力全面升级"

### 2. 封面图规范
- **尺寸**：建议900*383像素

- **内容**：包含版本号和核心功能图标

### 3. 排版规范

- **行间距**：1.5-1.8倍行间距
- **段落间距**：段落之间空一行
- **对齐方式**：左对齐
- **表情符号**：适当使用表情符号增强可读性
- **分隔线**：使用分割线区分不同部分

### 8. 图片获取与管理方案

#### 方案概述

为确保 Feat 版本发布文档和微信公众号文章的图片质量和稳定性，我们采用多种图片获取渠道和管理策略，避免对单一不稳定 API 的依赖。

### 8.1 图片获取渠道

#### 官方资源库
- **品牌素材库**：与设计团队合作创建，包含logo、图标、配色方案等
- **版本专用图片**：每个版本发布前 2 周准备，包括封面图、功能示意图等
- **存储位置**：`feat/images/` 目录，按类型和版本号组织

#### 设计团队协作
- **需求提交**：提前 2 周向设计团队提交图片需求
- **设计制作**：设计团队制作符合品牌风格的图片
- **审核确认**：文档编辑员审核图片质量和风格

#### 免费图库资源
- **推荐图库**：Unsplash、Pexels、Pixabay、Freepik
- **使用准则**：确保版权、风格匹配、分辨率足够

#### 截图资源
- **产品截图**：使用 Feat 实际运行界面的截图
- **代码可视化**：使用 Carbon、CodeSnap、Polacode 生成代码截图

#### 应急图片获取
- **快速生成**：Canva、Figma、PowerPoint
- **备用资源**：图标库、插图库、应急联系人

### 10. 常见插图问题及解决方案

#### 图片模糊

- **原因**：图片分辨率不足或被压缩过度
- **解决方案**：使用高分辨率图片，适当压缩

#### 加载缓慢

- **原因**：图片过大或网络问题
- **解决方案**：压缩图片，使用适当的格式

#### 版权纠纷

- **原因**：使用了无授权的图片
- **解决方案**：使用原创图片或获得授权的图片

#### 风格不一致

- **原因**：图片风格与品牌不符
- **解决方案**：统一图片风格，使用品牌色系

#### 图片插入失败

- **原因**：图片URL无效或格式错误
- **解决方案**：检查图片URL是否正确，确保图片可以正常访问

#### 图片显示异常

- **原因**：Markdown语法错误或图片路径问题
- **解决方案**：检查Markdown语法，确保图片路径正确

#### 图片尺寸不合适

- **原因**：图片尺寸与文档布局不匹配
- **解决方案**：使用HTML标签指定图片尺寸，或调整图片大小
