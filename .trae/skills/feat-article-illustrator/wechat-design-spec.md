# 微信公众号插图设计规范

## 一、微信生态视觉风格指南

### 1.1 色彩系统

#### 主色调规范

微信平台用户习惯于简洁、专业的视觉体验，色彩系统应遵循以下原则：

**品牌主色（Primary）**
```css
--feat-primary: #2563eb;      /* Feat 品牌蓝 - 专业、科技 */
--feat-primary-light: #3b82f6; /* 浅蓝变体 */
--feat-primary-dark: #1d4ed8;  /* 深蓝变体 */
```

**辅助色系统**
```css
--feat-secondary: #64748b;    /* 中性灰 - 稳重、专业 */
--feat-accent: #f59e0b;       /* 强调橙 - 活力、重点 */
--feat-success: #10b981;      /* 成功绿 - 积极、确认 */
--feat-warning: #f97316;      /* 警告橙 - 注意、提醒 */
--feat-error: #ef4444;        /* 错误红 - 重要、警示 */
```

**中性色系统**
```css
--neutral-50: #fafafa;        /* 最浅背景 */
--neutral-100: #f5f5f5;       /* 浅背景 */
--neutral-200: #e5e5e5;       /* 边框、分割线 */
--neutral-300: #d4d4d4;       /* 禁用边框 */
--neutral-400: #a3a3a3;       /* 占位文字 */
--neutral-500: #737373;       /* 辅助文字 */
--neutral-600: #525252;       /* 次要文字 */
--neutral-700: #404040;       /* 正文文字 */
--neutral-800: #262626;       /* 标题文字 */
--neutral-900: #171717;       /* 最深文字 */
```

#### 色彩应用原则

**60-30-10 法则适配微信**
- 主色 60%：背景、大面积色块（白色或浅灰）
- 辅助色 30%：次要元素、边框、分割线
- 强调色 10%：重点突出、按钮、图标

**微信深色模式适配**
```css
@media (prefers-color-scheme: dark) {
    --neutral-50: #171717;
    --neutral-100: #262626;
    --neutral-800: #e5e5e5;
    --neutral-900: #fafafa;
}
```

#### 不同文章类型配色方案

| 文章类型 | 主色 | 辅助色 | 强调色 | 适用场景 |
|----------|------|--------|--------|----------|
| 技术教程 | #2563eb | #64748b | #10b981 | 代码、架构、流程 |
| 产品发布 | #7c3aed | #64748b | #f59e0b | 新功能、版本更新 |
| 行业资讯 | #0891b2 | #64748b | #ef4444 | 新闻、动态、趋势 |
| 观点评论 | #dc2626 | #64748b | #f59e0b | 分析、见解、评论 |
| 故事案例 | #059669 | #64748b | #f97316 | 案例、经验、故事 |
| 活动推广 | #db2777 | #64748b | #8b5cf6 | 活动、促销、通知 |

### 1.2 字体规范

#### 字体层级系统

**标题层级**
```css
.title-h1 {
    font-size: 48px;
    font-weight: 700;
    line-height: 1.2;
    letter-spacing: -0.02em;
}

.title-h2 {
    font-size: 36px;
    font-weight: 600;
    line-height: 1.3;
    letter-spacing: -0.01em;
}

.title-h3 {
    font-size: 28px;
    font-weight: 600;
    line-height: 1.4;
    letter-spacing: 0;
}
```

**正文层级**
```css
.text-body-large {
    font-size: 18px;
    font-weight: 400;
    line-height: 1.6;
}

.text-body {
    font-size: 16px;
    font-weight: 400;
    line-height: 1.7;
}

.text-body-small {
    font-size: 14px;
    font-weight: 400;
    line-height: 1.6;
}
```

**辅助文字**
```css
.text-caption {
    font-size: 12px;
    font-weight: 400;
    line-height: 1.5;
    color: var(--neutral-500);
}

.text-overline {
    font-size: 11px;
    font-weight: 500;
    line-height: 1.5;
    letter-spacing: 0.05em;
    text-transform: uppercase;
}
```

#### 微信平台字体栈

```css
--font-stack: 
    -apple-system, 
    BlinkMacSystemFont, 
    "PingFang SC",        /* 苹方 - iOS/macOS 中文 */
    "Hiragino Sans GB",   /* 冬青黑体 - macOS 中文 */
    "Microsoft YaHei",    /* 微软雅黑 - Windows 中文 */
    "WenQuanYi Micro Hei",/* 文泉驿微米黑 - Linux 中文 */
    "Helvetica Neue",
    "Segoe UI",
    Roboto,
    sans-serif;
```

#### 字体使用原则

1. **可读性优先**：正文最小 14px，移动端建议 16px
2. **层级分明**：相邻层级字号差至少 4px
3. **字重对比**：标题 600-700，正文 400，辅助 300-400
4. **行高适配**：标题 1.2-1.4，正文 1.6-1.8

### 1.3 构图原则

#### 微信封面图构图

**大图封面（900×383）**
```
┌──────────────────────────────────────────────────────────────┐
│  [留白区域 60px]                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │                                                        │  │
│  │                    主视觉区域                           │  │
│  │                  （居中/左对齐）                         │  │
│  │                                                        │  │
│  │    标题文字（最大48px，建议1-2行）                       │  │
│  │    副标题/描述（16-18px）                               │  │
│  │                                                        │  │
│  └────────────────────────────────────────────────────────┘  │
│  [底部留白 40px]                    [品牌标识/Logo 位置]      │
└──────────────────────────────────────────────────────────────┘
```

**小图封面（200×200）**
```
┌────────────────────┐
│                    │
│    [图标/图形]      │
│      居中          │
│                    │
│    [简短标题]       │
│    1-4字为宜        │
│                    │
└────────────────────┘
```

**文中插图构图**

```
┌──────────────────────────────────────────────────────────────┐
│  [顶部留白 40px]                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  标题区域（可选）                                        │  │
│  └────────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────────┐  │
│  │                                                        │  │
│  │                    内容区域                             │  │
│  │              （图表/流程/概念图）                        │  │
│  │                                                        │  │
│  └────────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  说明文字/图注（可选）                                   │  │
│  └────────────────────────────────────────────────────────┘  │
│  [底部留白 40px]                                              │
└──────────────────────────────────────────────────────────────┘
```

#### 构图黄金法则

1. **三分法则**：将画面分为 9 等份，重要元素放在交叉点
2. **视觉引导**：使用线条、箭头引导视线流向
3. **留白呼吸**：边距至少 40px，元素间距 20-30px
4. **焦点突出**：每张图只突出一个核心信息

## 二、插图尺寸标准

### 2.1 微信公众号图片规格

| 图片类型 | 尺寸（像素） | 宽高比 | 文件大小 | 使用场景 |
|----------|--------------|--------|----------|----------|
| 封面大图 | 900 × 383 | 2.35:1 | ≤2MB | 图文消息首条封面 |
| 封面次图 | 900 × 500 | 1.8:1 | ≤2MB | 图文消息第二条及以后 |
| 封面小图 | 200 × 200 | 1:1 | ≤500KB | 公众号列表展示 |
| 文中插图 | 900 × 自适应 | - | ≤1MB | 文章内容配图 |
| 头图 | 900 × 300 | 3:1 | ≤500KB | 文章顶部品牌展示 |
| 尾图 | 900 × 200 | 4.5:1 | ≤300KB | 文章底部二维码/引导 |
| 分享图 | 500 × 400 | 1.25:1 | ≤500KB | 朋友圈分享缩略图 |

### 2.2 设备适配标准

**移动端优先原则**
- 设计基准：iPhone 14 Pro Max (430×932)
- 最小可读字号：14px
- 点击区域最小：44×44px

**多设备测试清单**
- [ ] iPhone SE (375×667)
- [ ] iPhone 14 (390×844)
- [ ] iPhone 14 Pro Max (430×932)
- [ ] Android 标准 (360×800)
- [ ] iPad Mini (768×1024)
- [ ] iPad Pro (1024×1366)

## 三、文章类型与插图模板

### 3.1 文章类型分类

#### 资讯类文章
**特征**：时效性强、信息密度高、客观中立
**插图风格**：简洁、信息清晰、数据可视化
**推荐元素**：时间轴、数据图表、关键信息卡片

## 四、文件命名与导出规范

### 4.1 文件命名规则

**命名格式**
```
{文章ID}_{插图类型}_{尺寸}_{序号}.{格式}
```

**命名元素说明**

| 元素 | 说明 | 示例 |
|------|------|------|
| 文章ID | 文章唯一标识，建议使用日期+序号 | 20240115_001 |
| 插图类型 | cover/inline/header/footer/share | cover |
| 尺寸 | 宽度×高度 | 900x383 |
| 序号 | 同类型多图时的序号 | 01, 02, 03 |

**命名示例**
```
20240115_001_cover_900x383_01.png    # 封面图
20240115_001_inline_900x600_01.png   # 文中插图1
20240115_001_inline_900x600_02.png   # 文中插图2
20240115_001_header_900x300_01.png   # 头图
20240115_001_share_500x400_01.png    # 分享图
```

### 4.2 导出格式规范

**格式选择**

| 图片类型 | 推荐格式 | 备选格式 | 说明 |
|----------|----------|----------|------|
| 封面图 | PNG | JPG | 有透明需求用PNG，照片类用JPG |
| 文中插图 | PNG | JPG | 图表类用PNG，照片类用JPG |
| 头图/尾图 | PNG | - | 通常包含品牌元素，建议PNG |
| 分享图 | JPG | PNG | 优先保证加载速度 |

**导出质量设置**

**PNG 格式**
- 色彩模式：RGB
- 位深度：24位（真彩色）或 32位（含透明通道）
- 压缩：无损压缩
- 适用：图标、图表、文字为主的图片

**JPG 格式**
- 色彩模式：RGB
- 质量：80-90%（平衡质量与大小）
- 渐进式：启用
- 适用：照片、复杂图像

### 4.3 文件大小控制

**压缩目标**

| 图片类型 | 目标大小 | 最大限制 | 压缩策略 |
|----------|----------|----------|----------|
| 封面大图 | ≤500KB | 2MB | TinyPNG 压缩 |
| 封面次图 | ≤400KB | 2MB | TinyPNG 压缩 |
| 文中插图 | ≤300KB | 1MB | 按需裁切、压缩 |
| 头图 | ≤200KB | 500KB | 简化设计 |
| 尾图 | ≤100KB | 300KB | 极简设计 |
| 分享图 | ≤200KB | 500KB | 适度压缩 |

**压缩工具推荐**
- TinyPNG：https://tinypng.com/
- Squoosh：https://squoosh.app/
- ImageOptim（macOS）

## 五、内容匹配度评估标准

### 5.1 评估维度

#### 主题相关性（权重：30%）

| 评分 | 标准 |
|------|------|
| 5分 | 插图完美契合文章主题，直观传达核心信息 |
| 4分 | 插图与主题高度相关，能有效辅助理解 |
| 3分 | 插图与主题相关，但关联度一般 |
| 2分 | 插图与主题关联较弱，需要文字说明 |
| 1分 | 插图与主题无关或产生误导 |

#### 视觉吸引力（权重：25%）

| 评分 | 标准 |
|------|------|
| 5分 | 视觉冲击力强，能立即吸引注意力 |
| 4分 | 视觉效果良好，有一定吸引力 |
| 3分 | 视觉效果一般，无特别亮点 |
| 2分 | 视觉效果较差，缺乏吸引力 |
| 1分 | 视觉效果差，影响阅读体验 |

#### 信息传达效率（权重：20%）

| 评分 | 标准 |
|------|------|
| 5分 | 信息传达清晰高效，一目了然 |
| 4分 | 信息传达较好，基本能快速理解 |
| 3分 | 信息传达一般，需要一定时间理解 |
| 2分 | 信息传达效率低，需要反复查看 |
| 1分 | 信息传达混乱，无法有效理解 |

#### 风格一致性（权重：15%）

| 评分 | 标准 |
|------|------|
| 5分 | 完全符合品牌风格，与系列文章高度统一 |
| 4分 | 基本符合品牌风格，风格较为统一 |
| 3分 | 风格基本协调，但有小幅偏差 |
| 2分 | 风格不够协调，有明显差异 |
| 1分 | 风格完全不协调，破坏整体体验 |

#### 技术规范符合度（权重：10%）

| 评分 | 标准 |
|------|------|
| 5分 | 完全符合尺寸、格式、大小等技术规范 |
| 4分 | 基本符合技术规范，有微小偏差 |
| 3分 | 部分符合技术规范，有可接受的偏差 |
| 2分 | 技术规范符合度较低，需要调整 |
| 1分 | 不符合技术规范，需要重新制作 |

### 5.2 综合评分标准

**计算公式**
```
总分 = 主题相关性 × 0.30 + 视觉吸引力 × 0.25 + 信息传达效率 × 0.20 
     + 风格一致性 × 0.15 + 技术规范符合度 × 0.10
```

**等级划分**

| 总分 | 等级 | 说明 | 处理建议 |
|------|------|------|----------|
| 4.5-5.0 | 优秀 | 完美插图，可直接使用 | 无需修改 |
| 3.5-4.4 | 良好 | 高质量插图，建议使用 | 微调优化 |
| 2.5-3.4 | 合格 | 基本可用，有改进空间 | 重点优化 |
| 1.5-2.4 | 待改进 | 质量不足，建议修改 | 重新设计 |
| 1.0-1.4 | 不合格 | 质量差，不建议使用 | 重新制作 |

### 5.3 评估检查清单

**使用前必检项**

- [ ] 插图是否与文章主题直接相关？
- [ ] 插图是否能在 3 秒内传达核心信息？
- [ ] 插图尺寸是否符合微信规范？
- [ ] 文件大小是否在限制范围内？
- [ ] 文字是否清晰可读（最小 14px）？
- [ ] 色彩对比度是否足够（WCAG AA 标准）？
- [ ] 是否包含品牌元素（Logo/名称）？
- [ ] 是否与系列文章风格一致？
- [ ] 是否避免了敏感/争议内容？
- [ ] 是否在移动端测试过显示效果？

## 六、微信平台特殊要求

### 6.1 内容规范

**禁止内容**
- 涉及政治敏感话题
- 包含违法违规信息
- 侵犯他人知识产权
- 含有虚假宣传内容
- 涉及低俗暴力元素

**注意事项**
- 人物肖像需获得授权
- 商标 Logo 需获得许可
- 数据来源需标注清楚
- 引用内容需注明出处

### 6.2 传播优化建议

**提升点击率**
- 封面图突出核心价值点
- 使用对比色增强视觉冲击
- 添加数字、符号等吸引元素
- 保持简洁，避免信息过载

**提升分享率**
- 设计易于传播的金句卡片
- 添加社交货币元素（如数据、排名）
- 制作可单独传播的信息图
- 考虑朋友圈分享场景

**提升完读率**
- 文中插图辅助理解复杂内容
- 合理控制插图数量（每 500 字 1-2 张）
- 插图与文字紧密配合
- 避免插图打断阅读节奏

### 6.3 无障碍设计

**色彩对比**
- 文字与背景对比度 ≥ 4.5:1
- 大文字对比度 ≥ 3:1
- 不仅依靠颜色传达信息

**文字可读**
- 正文最小 14px
- 行高 1.6-1.8
- 避免纯斜体或纯装饰字体

**替代文本**
- 为每张图片准备描述性 alt 文本
- 描述应包含图片核心信息
- 长度控制在 125 字符内

## 七、模板使用指南

### 7.1 模板选择流程

```
开始
  │
  ├─ 确定文章类型
  │   ├─ 资讯类 → 资讯模板
  │   ├─ 观点类 → 观点模板
  │   ├─ 故事类 → 故事模板
  │   ├─ 技术教程 → 教程模板
  │   ├─ 产品发布 → 发布模板
  │   └─ 活动推广 → 活动模板
  │
  ├─ 确定插图位置
  │   ├─ 封面 → 封面模板（900×383）
  │   ├─ 文中 → 文中模板（900×自适应）
  │   ├─ 头部 → 头图模板（900×300）
  │   └─ 底部 → 尾图模板（900×200）
  │
  ├─ 选择配色方案
  │   └─ 根据文章类型选择对应配色
  │
  └─ 填充内容
      ├─ 替换标题文字
      ├─ 替换内容区域
      ├─ 调整品牌元素
      └─ 导出并检查
```

### 7.2 快速修改指南

**修改配色**
1. 找到 `:root` 中的 CSS 变量
2. 修改 `--feat-primary` 等颜色值
3. 所有使用该变量的元素自动更新

**修改尺寸**
1. 修改 `.container` 的 `width` 和 `height`
2. 调整内部元素的 `padding` 和 `margin`
3. 确保文字大小适配新尺寸

**修改字体**
1. 修改 `body` 的 `font-family`
2. 调整 `font-size` 层级
3. 检查行高和字间距
