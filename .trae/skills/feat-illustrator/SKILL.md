# Feat 插图生成专家

## 重要说明

**最终输出格式：PNG/JPG 图片（通过截图生成）**

- 不要添加动画效果（animation、transition 等）
- 不要添加交互效果（hover、click 等）
- 专注于静态视觉效果
- 确保截图时的视觉呈现最佳

## 角色定义

Feat 插图生成专家负责根据文章内容生成专业级静态插图，通过HTML生成高质量图片，为Feat项目的文档、教程和文章提供视觉支持。

**核心能力**：内容分析 → 视觉翻译 → 风格匹配 → 精准执行

---

## 核心方法论：内容驱动设计四步法

### Step 1: 内容解码（Decode）

在开始设计前，必须回答：

| 维度 | 问题 | 输出 |
|------|------|------|
| What | 核心信息是什么？ | 关键词列表（≤5个） |
| Why | 图片的目的是什么？ | 信息传达/情感共鸣/行动引导 |
| Who | 受众是谁？ | 技术人员/普通用户/决策者 |
| Where | 图片出现在哪里？ | 封面/正文/结尾 |
| How | 如何表达最有效？ | 图表/插画/文字排版 |

### Step 2: 视觉翻译（Translate）

将抽象内容转化为视觉语言：

| 内容类型 | 视觉表达 | 推荐模板 |
|---------|---------|---------|
| 流程/步骤 | 线性排列 + 箭头 | art-growth.html |
| 对比/选择 | 左右分栏 + VS | art-compare.html |
| 层次/结构 | 堆叠/嵌套 | art-layers.html |
| 成长/变化 | 时间轴/曲线 | art-growth.html |
| 概念/抽象 | 图标 + 关键词 | art-concept.html |
| 情感/氛围 | 场景 + 色调 | art-mood.html |

### Step 3: 风格匹配（Match）

```
内容属性 → 风格选择决策树

                    内容是什么？
                         │
    ┌────────────────────┼────────────────────┐
    ↓                    ↓                    ↓
技术概念              产品功能              故事情感
    │                    │                    │
    ↓                    ↓                    ↓
简约专业              简约+手绘             手绘艺术
    │                    │                    │
    ↓                    ↓                    ↓
架构图/流程图         功能展示/对比图       概念图/氛围图
```

**风格特征对比**：

| 特征 | 简约专业风格 | 手绘艺术风格 |
|------|-------------|-------------|
| 线条 | 几何、规整 | 不规则、略带抖动 |
| 色彩 | 中性色为主，少量强调 | 暖色调，低饱和度 |
| 形状 | 几何形状 | 有机曲线 |
| 适用 | 技术教程、架构图 | 概念可视化、故事叙述 |

### Step 4: 精准执行（Execute）

1. **模板选择** → 根据内容类型选择模板
2. **内容填充** → 遵守内容约束（见下文）
3. **细节打磨** → 检查对齐、间距、层次
4. **质量检查** → 使用检查清单验证

---

## 一图一原则

每张插图必须遵循：

1. **单一焦点**：一张图只传达一个核心信息
2. **三秒法则**：读者3秒内应理解图片主旨
3. **视觉层级**：主标题 > 副标题 > 说明文字
4. **留白呼吸**：内容占比不超过60%
5. **品牌一致**：保持Feat视觉识别系统

## 内容约束规范

**固定容器特性**：所有模板使用 `overflow: hidden`，超出内容会被裁剪。

### 安全上限

| 内容类型 | 建议上限 | 说明 |
|----------|----------|------|
| 标题文字 | ≤12字符 | 主标题建议单行 |
| 副标题文字 | ≤20字符 | 可适当换行 |
| 列表项（垂直） | ≤4个 | 考虑间距和高度 |
| 列表项（水平） | ≤4个 | 考虑宽度和间距 |
| 每项文字 | ≤10字符 | 列表项内的文字 |
| 关键词标签 | ≤5个 | 水平排列时 |

### 防溢出检查

生成HTML前必须检查：
- [ ] 列表项数量是否在安全上限内
- [ ] 文字长度是否简洁
- [ ] 图标是否使用单字符 emoji
- [ ] 是否预留了足够的间距

**溢出处理**：精简内容 → 缩短文字 → 合并相似项 → 拆分多图

## 配色方案速查

### 简约专业风格

```css
:root {
    --primary: #1a1a1a;
    --secondary: #6b7280;
    --accent: #3b82f6;
    --neutral-light: #ffffff;
    --neutral-mid: #e5e7eb;
    --neutral-dark: #374151;
}
```

### 手绘艺术风格（优化版）

```css
:root {
    --art-warm-cream: #fef9f3;
    --art-soft-pink: #fce4ec;
    --art-lavender: #f3e5f5;
    --art-mint: #e8f5e9;
    --art-peach: #fff3e0;
    --art-sky: #e3f2fd;
    
    --art-primary: #7c6ba0;
    --art-primary-light: #9b8ab8;
    --art-primary-dark: #5a4a7a;
    
    --art-secondary: #e8a4b8;
    --art-accent: #e89b5a;
    --art-highlight: #e85a5a;
    
    --art-line: #5d4e37;
    --art-line-light: #6b5b45;
}
```

**配色设计理念**：
- 主色 `#7c6ba0`：优雅紫，提升视觉存在感
- 辅助色 `#e8a4b8`：柔和粉，形成鲜明层次对比
- 强调色 `#e89b5a`：暖橙，与暖色调背景和谐融合
- 线条色 `#6b5b45`：深棕，确保 WCAG AA 对比度标准（≥4.5:1）

详细配色方案见 [wechat-design-spec.md](wechat-design-spec.md)

## 工作流程

### 1. 分析阶段

- 接收文章内容
- 使用5W1H框架解码内容
- 确定插图位置和类型
- 选择风格（简约专业/手绘艺术）

### 2. 生成阶段

- 选择模板
- 填充内容（遵守约束）
- 保存到指定目录

### 3. 交付阶段

- 提供HTML文件
- 提供使用说明

### 4. 创意扩展

当现有模板无法满足需求时：

1. **创意提案** → 说明现有模板局限性，提出新方案
2. **模板设计** → 定义风格、元素、色彩
3. **模板开发** → 创建HTML文件
4. **文档更新** → 注册到SKILL.md

**模板命名规范**：`{风格前缀}-{类型}.html`
