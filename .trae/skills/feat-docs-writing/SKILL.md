# Feat 文档写作规范

## 核心理念

**好的技术文档应该直接说清楚。**

- 不绕弯子，不堆砌辞藻
- 代码示例真实可运行
- 该详细时不吝啬，该简洁时不啰嗦

---

## 写作风格

### 语言风格

**直接说清楚**

```
❌ Router 就像一个聪明的交通警察，把每个请求引导到正确的目的地。

✅ Router 负责将 HTTP 请求分发到对应的处理器。
```

**避免空洞的描述**

```
❌ 这是一个强大而灵活的功能。

✅ 它可以同时处理上万个并发连接，内存占用不到 10MB。
```

### 结构灵活性

**模板只是参考，不是枷锁**

- 简单的内容，一篇文档足够
- 复杂的内容，拆分成系列文档
- 根据内容特点调整结构，不必拘泥于固定格式

**该长则长，该短则短**

```
❌ 为了凑字数，把简单的概念讲得很长

✅ 简单的概念一句话说清楚，复杂的概念配图配代码详细讲解
```

## 代码示例规范

### 来源要求

**必须来自真实项目**，禁止手写：

| 来源 | 路径 |
|------|------|
| demo 模块 | `demo/` |
| 测试模块 | `feat-test/` |

### JDK 8 兼容性

```java
// 禁止
var list = new ArrayList<String>();           // JDK 10+
String json = """{"name":"test"}""";          // JDK 13+
record User(String name) {}                    // JDK 14+

// 正确
List<String> list = new ArrayList<String>();
String json = "{\"name\":\"test\"}";
public class User { private String name; }
```

### 注释风格

**注释要解释"为什么"，不只是"是什么"**

```java
// ❌ 只说是什么
server.listen(8080);  // 监听8080端口

// ✅ 解释为什么
server.listen(8080);  // 开发环境用 8080，生产环境建议通过配置指定
```

### 异常处理

```java
// 禁止
catch (Exception e) {
    e.printStackTrace();
}

// 正确
catch (Exception e) {
    System.err.println("处理失败: " + e.getMessage());
}
```

## 内容组织

### 文档类型

根据读者需求选择合适的写法：

| 读者想要... | 写法 | 特点 |
|-------------|------|------|
| 学会使用 | 教程式 | 手把手引导，由浅入深 |
| 解决问题 | 方案式 | 问题 → 方案 → 代码 |
| 理解原理 | 讲解式 | 比喻 → 原理 → 代码片段 |
| 查阅 API | 参考式 | 简洁明了，表格为主 |

### 内容去重

**每个知识点只在一处详细讲解**

其他地方简要提及 + 链接引用：

```mdx
<!-- 详细讲解处 -->
## Router 路由组件
Router 负责将 HTTP 请求分发到对应的处理器...
（完整讲解）

<!-- 其他文档引用时 -->
使用 Router 处理路由，详见 [Router 路由组件](/feat/server/router/)。
```

## 格式规范

### Aside 组件使用

**用于真正需要突出的信息，不要滥用**

```mdx
<!-- ✅ 适合用 Aside -->
<Aside type="caution">
生产环境请务必修改默认密码。
</Aside>

<!-- ❌ 不适合用 Aside，直接写在正文 -->
配置文件位于 `conf/app.yml`。
```

### 代码块

指定语言，提高可读性：

````mdx
```java
// Java 代码
```

```bash
mvn compile
```

```yaml
server:
  port: 8080
```
````

### 链接

```mdx
[内部文档](/feat/cloud/controller/)
[外部链接](https://maven.apache.org/)
[源码链接](https://gitee.com/smartboot/feat/blob/master/xxx)
```

## 质量检查

### 必须做到

- [ ] 代码来自真实项目，可运行
- [ ] 代码 JDK 8 兼容
- [ ] 无重复内容（已搜索现有文档）
- [ ] 链接全部有效

### 尽量做到

- [ ] 语言自然流畅
- [ ] 结构适合内容特点
- [ ] 关键步骤有注释

## 常见问题

### 写作卡壳怎么办？

1. 先写代码示例，再配文字说明
2. 想象你在给同事讲解这个功能
3. 看看竞品文档怎么写的

### 内容太长怎么办？

1. 检查是否有重复内容，改为链接引用
2. 考虑拆分成多篇文档
3. 用表格、列表替代冗长的段落

### 概念难解释怎么办？

1. 用生活中的类比
2. 用对比说明（与 Spring Boot、Vert.x 对比）
3. 用图示辅助
4. 用代码演示

## 核心原则

1. **直接说清楚** - 不绕弯子，不堆砌辞藻
2. **代码真实可运行** - 从实际项目复制
3. **内容不重复** - 链接引用已有内容
4. **灵活不教条** - 模板是参考，不是枷锁
