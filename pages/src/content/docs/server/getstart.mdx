这篇教程只做一件事：让你在最短路径内跑起第一个 Feat HTTP 服务，并知道 Feat Core 的最小编程模型是什么。

## 你会完成什么

- 在 Maven 项目中引入 `feat-core`
- 启动一个监听 `8080` 端口的最小 HTTP 服务
- 返回默认响应和自定义响应
- 知道接下来应该阅读哪几篇文档继续深入

## 前置条件

- JDK 1.8 或更高版本
- Maven 3.0 或更高版本
- 任意 Java IDE 或命令行环境
- [curl](https://curl.se/) 或浏览器，用于验证服务输出

:::note
如果你希望获得更接近 Spring Boot 的注解式开发体验，可以直接阅读 [Feat Cloud 快速入门](/feat/cloud/)。本页聚焦的是 Feat 最底层、最直接的 HTTP 服务编程方式。
:::

## 步骤 1：引入依赖

在 `pom.xml` 中添加 `feat-core`：

```xml title="pom.xml"
<dependency>
    <groupId>tech.smartboot.feat</groupId>
    <artifactId>feat-core</artifactId>
    <version>${feat.version}</version>
</dependency>
```

<Aside>
`${feat.version}` 请替换为实际版本，建议使用 Maven Central 上的[最新版本](https://central.sonatype.com/artifact/tech.smartboot.feat/feat-core)。
</Aside>

## 步骤 2：启动最小 HTTP 服务

创建 `HelloFeat.java`：

```java title="HelloFeat.java"
import tech.smartboot.feat.Feat;

public class HelloFeat {
    public static void main(String[] args) {
        Feat.httpServer().listen();
    }
}
```

这段代码已经完成了一个可工作的 HTTP 服务启动过程：

- `Feat.httpServer()` 创建 HTTP 服务器
- `listen()` 启动监听，默认端口是 `8080`
- 如果你没有注册自定义处理器，Feat 会返回一个默认响应，便于你先验证链路

## 步骤 3：验证服务是否启动成功

程序启动后，打开浏览器访问 `http://localhost:8080`，或者直接用 `curl`：

## 步骤 4：返回自定义响应

默认响应只适合做连通性验证。接下来给服务器加一个明确的请求处理器，并改用 `8081` 端口，避免与上一步同时运行时冲突。

```java title="HelloWorld.java"
import tech.smartboot.feat.Feat;

public class HelloWorld {
    public static void main(String[] args) {
        Feat.httpServer()
                .httpHandler(request -> request.getResponse().write("Hello World"))
                .listen(8081);
    }
}
```

这个例子里有两个最关键的 API：

- `httpHandler(...)`：注册请求处理逻辑
- `listen(8081)`：指定监听端口

也就是说，Feat Core 的最小编程模型就是：

1. 创建服务器
2. 注册处理器
3. 启动监听

## 常见问题

### 8080 或 8081 端口已被占用

最直接的做法是换端口：

```java
Feat.httpServer().listen(9090);
```

### 程序能启动，但浏览器访问不到

优先检查这三件事：

1. 你访问的端口是否与 `listen(...)` 一致
2. 本机防火墙或安全软件是否拦截了端口
3. 程序是否在启动后马上退出或报错

### 想进一步定制端口、线程数、调试模式

不要继续在 quickstart 里堆配置，直接进入 [ServerOptions 配置](/feat/server/serveroptions/)。
