Feat Core 服务端适合需要直接操作 HTTP 协议、精确控制每个字节，或者希望从零开始理解 Feat 服务端模型的开发者。

如果你希望用更接近 Spring Boot 的注解式开发体验，建议从 [Feat Cloud 快速开始](/feat/cloud/) 开始。

## 编写第一个应用

你将构建一个最小的 HTTP 服务，访问 `http://localhost:8080` 返回 `Hello Feat`。

**前置要求**：JDK 8+、Maven 3.0+、`curl` 或浏览器

### 1. 添加依赖

在 `pom.xml` 中加入：

```xml title="pom.xml"
<dependency>
    <groupId>tech.smartboot.feat</groupId>
    <artifactId>feat-core</artifactId>
    <version>${feat.version}</version>
</dependency>
```

<Aside>
`${feat.version}` 请替换为实际版本。最新版本可以在 [Maven Central](https://central.sonatype.com/artifact/tech.smartboot.feat/feat-core) 查看。
</Aside>

### 2. 编写服务

创建 `HelloFeat.java`：

```java title="HelloFeat.java"
import tech.smartboot.feat.Feat;

public class HelloFeat {
    public static void main(String[] args) {
        Feat.httpServer().listen();
    }
}
```

关键点：`Feat.httpServer()` 创建服务，`.listen()` 启动监听。

### 3. 运行并验证

运行 `HelloFeat.main()`，访问 `http://localhost:8080`：

### 自定义响应

接管请求处理，返回自定义响应：

```java title="HelloWorld.java"
import tech.smartboot.feat.Feat;

public class HelloWorld {
    public static void main(String[] args) {
        Feat.httpServer()
            .httpHandler(request -> request.getResponse().write("Hello World"))
            .listen(8081);
    }
}
```

Feat Core 的模型：创建服务 → 注册处理器 → 启动监听。

## 常见问题

**端口被占用**
```java
Feat.httpServer().listen(9090);
```

**浏览器访问不到**
1. 端口是否一致
2. 程序是否退出或报错
3. 防火墙是否拦截
