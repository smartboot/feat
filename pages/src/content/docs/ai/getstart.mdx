这篇教程只关注最小闭环：让模型真正返回结果。先把链路打通，再进入对话配置、Agent 和 Embedding。

## 你将完成什么

- 在 Maven 项目中引入 `feat-ai`
- 连接一个可用模型服务
- 运行第一次同步或流式对话
- 知道接下来应该去哪一页继续深入

## 前置条件

- JDK 1.8 或更高版本
- Maven 3.0 或更高版本
- 任选一种模型来源
  - 本地模型：已安装并运行 [Ollama](https://ollama.ai/)
  - 云端模型：已准备好 `FEAT_AI_API_KEY`

<Aside type="tip">
如果你只是想先验证链路是否打通，优先使用 Ollama。本地依赖更少，排查路径也更短。
</Aside>

## 步骤 1：引入依赖

在 `pom.xml` 中添加 `feat-ai`：

```xml title="pom.xml"
<dependency>
    <groupId>tech.smartboot.feat</groupId>
    <artifactId>feat-ai</artifactId>
    <version>${feat.version}</version>
</dependency>
```

<Aside>
`feat-ai` 已传递依赖 `feat-core`，通常无需再单独引入。
</Aside>

## 步骤 2：选择你的第一个模型来源

<Tabs>
    <TabItem value="Ollama" label="Ollama">
        ```java title="HelloFeatAI.java"
        import tech.smartboot.feat.ai.FeatAI;
        import tech.smartboot.feat.ai.chat.ChatModel;

        public class HelloFeatAI {
            public static void main(String[] args) {
                ChatModel model = FeatAI.chatModel(opts -> opts
                        .baseUrl("http://localhost:11434/v1")
                        .model("qwen2.5:7b")
                );

                model.chatStream("用一句话介绍 Feat 框架", System.out::print);
            }
        }
        ```

        运行前先确保本地模型已准备好，例如先执行：

        ```shell
        ollama run qwen2.5:7b
        ```
    </TabItem>
    <TabItem value="Gitee AI" label="Gitee AI">
        ```java title="HelloFeatAI.java"
        import tech.smartboot.feat.ai.FeatAI;
        import tech.smartboot.feat.ai.chat.ChatModel;
        import tech.smartboot.feat.ai.chat.ChatModelVendor;

        public class HelloFeatAI {
            public static void main(String[] args) {
                ChatModel model = FeatAI.chatModel(opts -> opts
                        .model(ChatModelVendor.GiteeAI.Qwen2_5_72B_Instruct)

                model.chat("用一句话介绍 Feat 框架", response -> {
                    System.out.println(response.getContent());
                });
            }
        }
        ```

        运行前请先设置环境变量 `FEAT_AI_API_KEY`，否则模型初始化会失败。
    </TabItem>

## 步骤 3：验证模型确实返回了结果

程序运行成功后，你应该能看到模型返回的文本内容。内容不必与示例完全一致，但至少应该满足以下预期：

- 程序没有抛异常
- 控制台有真实文本输出，而不是空内容
- 你能区分“同步一次性输出”和“流式逐段输出”

一个典型的输出可能类似：

```text
Feat 是一个轻量级、高性能的 Java Web 服务开发框架。
```

<Aside type="caution">
使用云端模型时，环境变量未设置通常会直接导致初始化失败，而不是返回空内容。
</Aside>

## 步骤 4：理解同步调用和流式调用的区别

在 quickstart 阶段，你只需要记住下面这个区别：

- `chat(...)`：拿到完整回复后再处理，适合后台任务或一次性返回
- `chatStream(...)`：边生成边消费，适合 CLI 输出、前端打字机效果、长文本生成

当最小示例跑通后，你通常会继续做这三件事：

1. 给模型增加 `system(...)`、`temperature(...)`、`maxTokens(...)` 等配置。
2. 从一次性输出切换到 `chatStream(...)`，把内容实时返回给前端。
3. 维护消息列表，构建多轮对话上下文。

这些能力在 [对话与流式](/feat/ai/chat/) 中有完整示例。

## 常见问题

### Ollama 模式下没有任何输出

优先检查：

1. `ollama` 服务是否已经运行
2. 本地是否真的有你指定的模型，比如 `qwen2.5:7b`
3. `baseUrl` 是否与你本机环境一致，默认一般是 `http://localhost:11434/v1`

### Gitee AI 模式下程序直接报错

优先检查 `FEAT_AI_API_KEY` 是否已设置到当前运行环境中，而不是只设置在某个终端会话里。
