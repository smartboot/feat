这篇教程的目标不是覆盖全部注解能力，而是让你在最短路径内确认三件事：

- `feat-cloud-starter` 依赖已经引入成功
- `@Controller` 和 `@RequestMapping` 已经生效
- Feat Cloud 能够扫描代码并启动一个可访问的 HTTP 服务

## 你会完成什么

- 创建一个最小的 Feat Cloud Maven 项目
- 编写一个返回字符串的控制器
- 启动服务并访问 `/hello`
- 了解遇到注解处理问题时应该怎么排查

## 前置条件

- JDK 1.8 或更高版本
- Maven 3.0 或更高版本
- IntelliJ IDEA 或任意支持 Maven 的 Java IDE

## 步骤 1：引入依赖

创建一个 Maven 项目，然后在 `pom.xml` 中添加 `feat-cloud-starter`：

```xml title="pom.xml"
<dependency>
    <groupId>tech.smartboot.feat</groupId>
    <artifactId>feat-cloud-starter</artifactId>
    <version>${feat.version}</version>
</dependency>
```

<Aside title="关于版本号">
`${feat.version}` 请替换为实际版本，建议使用 Maven Central 上的[最新版本](https://central.sonatype.com/artifact/tech.smartboot.feat/feat-cloud-starter)。
</Aside>

## 步骤 2：编写最小控制器

在 `src/main/java` 下创建 `Bootstrap.java`：

```java title="Bootstrap.java"
import tech.smartboot.feat.cloud.FeatCloud;
import tech.smartboot.feat.cloud.annotation.Controller;
import tech.smartboot.feat.cloud.annotation.RequestMapping;

@Controller
public class Bootstrap {

    @RequestMapping("/hello")
    public String helloWorld() {
        return "hello Feat Cloud";
    }

    public static void main(String[] args) {
        FeatCloud.cloudServer().listen();
    }
}
```

这个例子已经覆盖了 Feat Cloud 的核心启动链路：

- `@Controller`：声明控制器
- `@RequestMapping("/hello")`：声明路由映射
- `FeatCloud.cloudServer().listen()`：启动 Feat Cloud 并监听请求

## 步骤 3：启动应用

直接运行 `Bootstrap.main()`。当控制台里出现路由注册和监听地址时，就说明应用已经成功启动。

一个典型的启动输出会包含类似内容：

```text
Feat Router:
 |-> /hello ==> Bootstrap@helloWorld
http://0.0.0.0:8080/
```

如果你能看到 `/hello ==> Bootstrap@helloWorld` 这一类映射信息，说明控制器扫描和注解处理都已生效。

## 步骤 4：验证路由是否真正可访问

启动成功后，访问 `http://localhost:8080/hello`：

## 常见问题

### 控制台启动了，但访问 `/hello` 返回 404

优先检查：

1. 你的类是否真的加了 `@Controller`
2. 方法是否加了 `@RequestMapping("/hello")`
3. IDE 是否正确触发了注解处理和增量编译

### IntelliJ IDEA 下修改代码后，路由没有更新

旧版本 IDEA 或某些增量编译场景下，APT 生成文件可能不会及时刷新。此时建议：

1. 打开 `Preferences`
2. 进入 `Build, Execution, Deployment -> Build Tools -> Maven -> Runner`
3. 勾选 `Delegate IDE build/run actions to Maven`

<img src={idea_setting.src} alt="IDEA 配置截图" width="60%" className="shadow"/>

<Aside type="tip">
如果你确认本地环境会频繁遇到 APT 失效，再考虑在 `pom.xml` 中加入 `maven-clean-plugin`。否则不要在 quickstart 阶段先引入额外复杂度。
</Aside>
