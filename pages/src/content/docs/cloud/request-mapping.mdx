上一章写出了一个能返回文本和 JSON 的控制器。这一章继续扩展它，学习如何接收客户端传过来的参数，以及如何为同一路径配置不同的 HTTP 方法。

HTTP 接口真正进入业务之后，固定返回值很快就不够用了。你需要从查询字符串里拿筛选条件，从路径里拿资源标识，从 JSON 请求体里拿业务对象，再用不同 HTTP 方法表达读取、创建、更新和删除。

## 查询参数

使用 `@Param` 注解从 URL 查询字符串提取参数值。

```java title="UserController.java"
@RequestMapping(value = "/search", method = RequestMethod.GET)
public String search(@Param("keyword") String keyword,
                     @Param("page") int page) {
    return "搜索: " + keyword + ", 页码: " + page;
}
```

### 支持的参数类型

| 类型类别 | 具体类型 | 说明 |
|---------|---------|------|
| 基本类型 | `int`、`long`、`boolean` 等 | 缺失参数时取值为 `null`，拆箱可能触发 `NullPointerException` |
| 包装类型 | `Integer`、`Long`、`Boolean` 等 | 缺失参数时为 `null` |
| 字符串 | `String` | 缺失参数时为 `null` |

<Aside type="caution">
基本类型没有 `null` 状态。如果查询字符串里没有对应参数，内部解析会得到 `null`，再拆箱成 `int` 或 `long` 时就会抛出 `NullPointerException`。建议把可能缺失的参数声明为包装类型，或在业务代码里做判空处理。
</Aside>

## 路径参数

使用 `@PathParam` 注解从 URL 路径提取变量值。

```java title="UserController.java"
@RequestMapping(value = "/:userId", method = RequestMethod.GET)
public String getUser(@PathParam("userId") String userId) {
    return "用户ID: " + userId;
}
```

<Aside type="tip">
Feat Cloud 支持两种路径参数写法：

- 冒号形式：`/:userId`
- 花括号形式：`/{userId}`

两种写法功能等价，团队内部统一即可。
</Aside>

## JSON 请求体

创建和更新资源时，参数通常不会继续塞在 URL 里，而是放进 JSON 请求体。Controller 方法可以直接声明一个自定义 POJO 参数，不需要额外注解。

```java title="User.java"
public class User {
    private String username;
    private String password;
    private String role;

    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }

    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }

    public String getRole() { return role; }
    public void setRole(String role) { this.role = role; }
}
```

```java title="UserController.java"
@RequestMapping(value = "/", method = RequestMethod.POST)
public String create(User user) {
    return "created user: " + user.getUsername();
}
```

请求时带上 `Content-Type: application/json`：

```shell title="curl 请求"
curl -X POST http://localhost:8080/users/ \
  -H 'Content-Type: application/json' \
  -d '{"username":"Bob","password":"123456","role":"admin"}'
```

返回：

```text title="响应结果"
created user: Bob
```

这里的判断规则很简单：`@Param` 读取 URL 或表单参数，`@PathParam` 读取路径变量，自定义 POJO 参数读取 JSON 请求体。进入正式业务接口后，常见做法是用路径表达资源位置，用请求体承载要创建或更新的数据。

## HTTP 方法

### RequestMethod 枚举

```java title="RequestMethod.java"
public enum RequestMethod {
    GET, HEAD, POST, PUT, PATCH, DELETE, OPTIONS, TRACE
}
```

### RESTful 接口

为同一路径配置不同方法：

```java title="UserController.java"
@RequestMapping(value = "/", method = RequestMethod.GET)
public String list() {
    return "list all users";
}

@RequestMapping(value = "/", method = RequestMethod.POST)
public String create(User user) {
    return "created user: " + user.getUsername();
}

@RequestMapping(value = "/:id", method = RequestMethod.GET)
public String get(@PathParam("id") String id) {
    return "get user: " + id;
}

@RequestMapping(value = "/:id", method = RequestMethod.PUT)
public String update(@PathParam("id") String id, User user) {
    return "updated user " + id + " to " + user.getUsername();
}

@RequestMapping(value = "/:id", method = RequestMethod.DELETE)
public String delete(@PathParam("id") String id) {
    return "deleted user: " + id;
}
```

## 路径风格如何取舍

Feat Cloud 同时支持 `:id` 和 `{id}` 两种路径参数写法。  
如果你的团队来自前端路由或 Feat Core 路由体系，`:id` 会更自然；如果团队长期使用 Spring MVC，`{id}` 往往更容易迁移。

两者没有能力差异，真正重要的是团队内部保持一致。路径风格一旦混用，后续排查路由时会增加不必要的认知成本。
