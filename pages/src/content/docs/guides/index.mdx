Feat 是一个从底层网络服务能力生长出来的 Java Web 框架，把 Core、Cloud、AI 三条线收拢到同一套技术栈里。

它不是 Servlet 容器的包装层，也不是单纯的注解框架。

<img src={architecture.src} alt="Feat 架构图" width="800" height="400" />

| 层级 | 定位 | 适合场景 |
|------|------|----------|
| **Feat Core** | 底层 HTTP 服务 | 直接控制请求处理，关注性能和资源占用 |
| **Feat Cloud** | 注解式开发模型 | 快速构建业务应用，类似 Spring Boot 体验 |
| **Feat AI** | AI 能力接入层 | 统一接入对话、Embedding、Agent、MCP |

## 阅读建议

第一次接触 Feat 的读者，建议先选一条路径跑通一个最小示例，再沿着该路径继续深入。不需要一开始就读完所有文档。

- 对某个能力感兴趣 → 沿着该能力的教程继续读
- 需要查参数、排错 → 再进入配置页和附录

先跑通，再扩展；先教程，再参考。
