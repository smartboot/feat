如果你已经会用 Java 写 Web 服务，但希望有一个比 Spring Boot 更轻量、启动更快、资源占用更低的选项；或者你想把 AI 能力直接接入 Java 后端而不引入 Python 服务，Feat 很可能是你要找的工具。

Feat 是一个从底层网络服务能力生长出来的 Java Web 框架。它不是 Servlet 容器的包装层，也不是单纯的注解框架。它把 Core、Cloud、AI 三条线收拢到同一套技术栈里，每一层都可以独立使用。

<img src={architecture.src} alt="Feat 架构图" width="800" height="400" />

| 层级 | 定位 | 适合场景 |
|------|------|----------|
| **Feat Core** | 底层 HTTP 服务 | 直接控制请求处理，关注性能和资源占用 |
| **Feat Cloud** | 注解式开发模型 | 快速构建业务应用，类似 Spring Boot 体验 |
| **Feat AI** | AI 能力接入层 | 统一接入对话、Embedding、Agent、MCP |

<Aside type="caution">
Feat 主线仓库采用 Apache 2.0。Feat Core 与 Feat AI 可直接使用；Feat Cloud 通过赞助支持模式开放，如需使用请先[成为赞助者](/feat/sponsors/)获取授权。
</Aside>
