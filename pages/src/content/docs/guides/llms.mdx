Feat 文档已通过 [llms.txt](https://llmstxt.org/) 规范向 AI 工具开放。这意味着你可以在使用 Cursor、Windsurf、Claude Code 等工具时，直接引用 Feat 的官方文档，让 AI 在代码补全、错误排查和架构设计时给出更准确的建议。

## 什么是 llms.txt

[llms.txt](https://llmstxt.org/) 是一种面向大语言模型的文档格式标准。它把项目文档转换成 AI 更容易理解和引用的结构，帮助 AI 工具：

- 准确理解 Feat 的 API 设计和使用模式
- 在代码补全时遵循 Feat 的最佳实践
- 在排查问题时引用正确的文档章节

Feat 文档站点通过 `starlight-llms-txt` 插件自动生成并维护这些文件。

## 可用的文档资源

| 文件 | 地址 | 说明 |
|------|------|------|
| llms.txt | `https://smartboot.tech/feat/llms.txt` | 导航文件，包含所有文档的索引和链接 |
| llms-small.txt | `https://smartboot.tech/feat/llms-small.txt` | 精简版文档，适合上下文窗口有限的场景 |
| llms-full.txt | `https://smartboot.tech/feat/llms-full.txt` | 完整文档，适合需要全面了解 Feat 的场景 |

<Aside type="tip">
如果 AI 工具的上下文窗口有限（如 128K 或更小），建议优先使用 `llms-small.txt`，它只包含核心 API 和最常见的使用模式。
</Aside>

## 在 AI 工具中配置

### Cursor

<Steps>
1. 打开 Cursor 设置，进入 `Features > Docs`

2. 点击 "Add new doc"

3. 输入 Feat 文档地址：`https://smartboot.tech/feat/llms.txt`

4. 在对话中使用 `@feat` 引用文档上下文
</Steps>

### Windsurf

<Steps>
1. 在 Windsurf 中打开 "Add context" 面板

2. 选择 "Web" 或 "URL" 选项

3. 输入 `https://smartboot.tech/feat/llms.txt`

4. 在 Cascade 对话中通过 `@` 引用 Feat 文档
</Steps>

### Claude Code

<Steps>
1. 打开 Claude Code 设置，进入 "Docs / Context Files" 配置

2. 添加 Feat 文档地址：`https://smartboot.tech/feat/llms.txt`

3. 在代码补全和解释时，Claude 会自动引用其中的内容
</Steps>

### GitHub Copilot

<Steps>
1. 在 VS Code 中打开 Copilot Chat

2. 使用 `#fetch` 指令引入文档：
    ```
    #fetch https://smartboot.tech/feat/llms.txt
    ```

3. 后续提问中 Copilot 会参考 Feat 的文档上下文
</Steps>

### Qwen Code

<Steps>
1. 在 Qwen Code 中打开 "知识库" 或 "Context" 配置

2. 添加 Feat 文档地址：`https://smartboot.tech/feat/llms.txt`

3. 在代码生成和补全时，Qwen Code 会参考 Feat 文档给出建议
</Steps>

### Trae

<Steps>
1. 打开 Trae 设置，进入 `Settings > Context > Docs`

2. 添加 Feat 文档地址：`https://smartboot.tech/feat/llms.txt`

3. 在 Builder 或 Chat 中，Trae 会自动引用 Feat 文档上下文回答问题
</Steps>

## 使用建议

### 什么时候引用文档

- **初始化项目时**：让 AI 根据 Feat 的最佳实践生成项目结构
- **遇到报错时**：提供错误信息的同时引用文档，获得更准确的排查建议
- **选择组件时**：询问 AI 不同场景下应该使用 Feat 的哪个功能模块
- **性能优化时**：让 AI 参考文档中的性能建议给出优化方案

### 如何获得更好的回答

<Aside type="tip">
不要只问 "怎么用 Feat"。提供具体的场景和约束条件，AI 才能结合文档给出精准建议。
</Aside>

**推荐的提问方式：**

```
我正在用 Feat 开发一个 REST API 服务，需要：
1. 处理 JSON 请求和响应
2. 实现路由分发
3. 添加全局异常处理

请根据 @feat 文档给出完整的代码示例。
```

**避免的提问方式：**

```
怎么用 Feat 写 Web 服务？
```
